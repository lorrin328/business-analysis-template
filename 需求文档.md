## 1. 项目概述

### 1.1 业务背景

本项目服务于 **太平人寿保险有限公司网电多元条线**，覆盖以下主要业务线：

| 业务线 | 说明 |
|--------|------|
| **OTO** | 线上到线下融合业务 |
| **证保** | 证券渠道保险业务 |
| **蚁桥** | 蚂蚁渠道桥梁业务 |
| **线上经代** | 线上保险经纪/代理业务 |

**当前重点**：正在推动 **职域营销** 新模式，主要与 **团险渠道** 合作开展，目前处于 **试点阶段**。

**业务调整**：原电销业务已全部关停，本看板不再涉及传统电销数据。

网电多元条线需要一套轻量级的经营分析工具，覆盖业务、人力、产品、目标四个维度，用于：

- 周例会、周检会、季度经营分析会的数据汇报
- 机构、业务模式、产品维度的多视角钻取
- 同期对比、达成率追踪、结构演变监控
- 在没有 IT 后台支持的情况下，由企划同事独立运营

### 1.2 产品形态

**单文件 HTML，纯浏览器实现，零安装**：

- 双击 `经营分析模板.html` → 浏览器打开 → 选择 Excel → 自动入库 → 渲染
- 无需 Python、Node、服务器；所有计算在浏览器内完成
- 数据通过 IndexedDB 持久化，下次打开秒级恢复

### 1.3 当前能力

截至 2026-04-27（v1）：

- ✅ 顶部导航 + 4 大版块弹窗（关键 KPI / 趋势表现 / 产品结构 / 队伍表现）
- ✅ 保费数据（Excel）+ 目标数据（Excel）双源导入
- ✅ 三口径切换（折算 / 期交 / 规模），10+ 维度筛选，多年同比，日明细 hover
- ✅ 达成率、规模同比、产品结构饼图、队伍 7 项指标表
- ⏳ 模块化迁移中（5 个 ESM 模块占位，业务逻辑仍在内联 IIFE）
- ⏳ 周维度、KPI 导出、人力基表完整接入待开发

### 2.2 技术栈

| 组件 | 版本 | 用途 |
|------|------|------|
| ECharts | 5.5 | 折线图、饼图、柱状图 |
| sql.js | 1.10.3 | 浏览器内 SQLite（WASM） |
| SheetJS / XLSX | 0.18.5 | Excel 解析 |
| IndexedDB | 浏览器原生 | 二进制 db 持久化 |

所有依赖通过 jsDelivr CDN 引入，离线使用需替换为本地路径。

### 2.3 模块化形态（迁移中）

项目正在从「单文件巨型 IIFE」过渡到「ESM 源码 + build.sh 注入」的双形态：

- **开发期**：通过 HTTP 服务（如 `python -m http.server`）打开 → `<script type="module" src="js/main.js">` 加载 ESM；调试友好。
- **交付期**：执行 `build.sh` → 把 `js/` 全部 import 展开后注入到 `经营分析模板.html` 的 `<!-- BUILD:JS:CORE -->` 占位 → 输出 `dist/经营分析模板.html`，可双击 `file://` 打开。
- **过渡期**：ESM 模块与内联 IIFE 共存。`globalThis.__jyfx` 命名空间承载 ESM 模块，内联 IIFE 在同名局部声明上覆盖兜底。**当前 5 个新模块（modal/target-importer/kpi/team-performance/部分 platform-trend 弹窗逻辑）实际逻辑均在内联 IIFE 中**，ESM 仅为占位。

详见 [模块化重构总体方案 v0.2](./docs/需求/2026-04-26_模块化重构总体方案.md)。

### 2.4 启动流程

```
页面加载
  ├─ initSqlJs()                  # sql.js WASM 初始化
  ├─ openIDB() → loadKV('db')     # 尝试从 IndexedDB 恢复
  ├─ 有数据 → exec(SELECT) 渲染主界面
  └─ 无数据 → 显示欢迎页 + 上传引导
```

启动期间显示三态 overlay：`loading`（加载中） / `empty`（无数据） / `error`（解析失败）。

## 3. 数据模型

### 3.1 `fact_premium`（保单事实表）

承载所有保单明细，是绝大多数指标的来源。完整字段表见 [数据规范/保单数据规范.md](./docs/数据规范/保单数据规范.md)，关键列：

| 入库列 | 类型 | 说明 |
|--------|------|------|
| `year`, `quarter`, `month`, `day`, `date` | TEXT/INT | 时间分组键，按 **入账时间** 推导 |
| `org` | TEXT | 销售机构名称 |
| `biz_mode` | TEXT | 业务模式 |
| `term_type` | TEXT | 长短险（长期 / 短期；极短期合并入短期） |
| `pay_years`, `cov_years` | TEXT | 缴费年限 / 保障年限 |
| `is_annuity` | TEXT | 是否商保年金产品 |
| `design_cat` | TEXT | 产品设计分类（饼图主维度） |
| `staff_id` | TEXT | 人员工号（关联人力表） |
| `policy_no` | TEXT | 投保单号（件数统计） |
| `self_mark` | TEXT | 自保件标记（Y/N） |
| `qj_cents` | INTEGER | 期交保费（分） |
| `zhsf_cents` | INTEGER | 折算保费（分） |
| `ghgb_cents` | INTEGER | 年化规保（分） |

**金额一律按「分」存 INTEGER**：导入时 `元 × 100` 后 `Math.round()`，查询时 `SUM(col) / 100.0` 还原，避免浮点误差。

### 3.2 `fact_headcount`（人力事实表 - 规划）

承载月度人力快照，与保单表通过 `staff_id` 关联。详见 [数据规范/人力数据规范.md](./docs/数据规范/人力数据规范.md)：

| 入库列 | 类型 | 说明 |
|--------|------|------|
| `period_ym` | TEXT | `YYYY-MM` |
| `org` | TEXT | 销售机构名称 |
| `staff_id` | TEXT | 人员代码 |
| `sales_seq` | TEXT | 销售序列 |
| `rank_grade`, `rank_level` | TEXT | 职等 / 职级 |
| `tenure_band` | TEXT | 月末司龄区间（a-f 六档） |
| `biz_mode` | TEXT | 业务模式名称 |
| `hc_start`, `hc_end` | INTEGER | 月初 / 月末在职（0/1） |
| `mgmt_level` | TEXT | 派生：**外勤管理职** / **一线营销员** |

**约束**：`UNIQUE INDEX (period_ym, staff_id)`；增量导入采用 `DELETE WHERE period_ym=? + INSERT`。

**当前状态**：表结构已规划，导入流程未实装；现版本队伍表现指标基于 `fact_premium` 出单记录近似推算（见 §8 已知限制）。

### 3.3 `fact_target`（目标事实表）

承载用户上传的目标数据：

| 列 | 类型 | 说明 |
|----|------|------|
| `year` | TEXT | 必需 |
| `quarter` | TEXT | 季度（按维度填写） |
| `month` | TEXT | 月份（按维度填写） |
| `week` | TEXT | 周次（预留） |
| `org` | TEXT | 机构（按维度填写） |
| `project` | TEXT | 项目（按维度填写） |
| `metric` | TEXT | 指标名（折算保费 / 期交保费 / 规模保费） |
| `target_value` | REAL | 目标值（元） |

**写入规则**：上传同一年度目标会覆盖已有数据（`DELETE WHERE year=? + INSERT`）。

### 3.4 IndexedDB 存储

数据库 `business-analysis-db` → ObjectStore `kv`：

| Key | Value |
|-----|-------|
| `db` | sql.js 导出的 `Uint8Array`（完整 SQLite 二进制） |
| `meta` | `{ importedAt, fileName, rowCount, ... }` |

入库后写入 IndexedDB，下次打开自动恢复，无需重新解析 Excel。

## 4. 指标定义

> 完整 SQL 与边界处理见 [指标口径手册.md](./docs/指标口径手册.md)。

### 4.1 保费类（三口径）

| 指标 | 公式 | 入库列 |
|------|------|--------|
| **规模保费** | 各类产品的规模保费汇总 | `ghgb_cents` |
| **期交保费** | 非趸交长险 + 短险 + 长险趸交 × 0.1 | `qj_cents` |
| **折算保费 / 标准保费** | 长险趸交×0.1 + [3,5)年×0.3 + [5,10)年×0.5 + 10年及以上×1 + 短险×0.1 | `zhsf_cents` |

**口径切换**：UI 提供「折算 / 期交 / 规模」三按钮，所有图表与表格联动。默认口径：**折算保费**。

### 4.2 达成率

```
达成率 = 实际保费 / 目标保费
```

- 数据源：`fact_premium`（实际） + `fact_target`（目标）
- 时间维度：年度 / 季度 / 月度
- 颜色规则：≥100% 绿色，<100% 红色

### 4.3 同比（YoY）

```
yoy(year, period) = (current_cum - base_cum) / base_cum × 100%
base_year = new Date().getFullYear()
```

- **基准年自动滚动**：`BASE_YEAR = String(new Date().getFullYear())`，2027-01-01 起自动切到 2027，**不硬编码、不允许 UI 选择**。
- **规模同比**：当期累计 vs 上年同期累计的百分比变化。
- **活动率同比**：使用百分点差（pp），不使用百分比。

### 4.4 人力类

| 指标 | 公式 | 备注 |
|------|------|------|
| 月末在职人力 | `SUM(hc_end)` | 当前版本由 `COUNT(DISTINCT staff_id WHERE month=...)` 近似 |
| 月初在职人力 | `SUM(hc_start)` | 同上近似 |
| 月均在职人力 | `(月初 + 月末) / 2` | 简单平均 |
| 月均活动人力 | 当月有出单且在职的工号数 | `COUNT(DISTINCT staff_id WHERE 月度有出单)` |
| 13 个月留存率 | 入职 13 个月后仍在职人数 / 入职人数 | 待实现，需入离职日期 |

### 4.5 活动率

| 指标 | 公式 |
|------|------|
| 活动率 | 当月累计开单人力 / 月均在职人力 |
| **长险活动率** | 当月累计长险开单人力 / 月均在职人力 |

```sql
-- 长险活动率（v2 形态，依赖 fact_headcount）
SELECT
  (COUNT(DISTINCT CASE WHEN p.term_type='长期' THEN p.staff_id END) * 1.0)
  / (SUM(h.hc_start + h.hc_end) / 2.0) AS rate
FROM fact_headcount h
LEFT JOIN fact_premium p
  ON p.staff_id = h.staff_id AND substr(p.date, 1, 7) = h.period_ym
WHERE h.period_ym = ?;
```

### 4.6 产能

| 指标 | 公式 | 单位 |
|------|------|------|
| **P1 人均保费** | 累计保费 / 月均在职人力 | 元 |
| **P2 人均产能** | 累计保费 / 月均活动人力 | 元 |
| **P3 人均件数** | 累计件数 / 月均活动人力 | 件 |

**保费口径联动**：P1/P2 的「累计保费」跟随顶部三口径切换。

### 4.7 标准人力（队伍表现）

按业务模式动态阈值：

| 业务模式 | 阈值 |
|----------|------|
| OTO | 月度人均保费 ≥ 2 万元 |
| 证保 | 月度人均保费 ≥ 3 万元 |
| 其他 | 暂未定义 |

**现状**：阈值硬编码，待支持配置化。

### 4.8 质量类（预留）

| 指标 | 公式 |
|------|------|
| 13 个月继续率 | 第 13 个月有效保费 / 13 个月前应续保费 |
| 25 个月继续率 | 第 25 个月有效保费 / 25 个月前应续保费 |

未在看板中实现，需要续期数据源。

## 5. 业务逻辑

### 5.1 时间口径

- **保费统计时间**：按 **入账时间** 推导 `年/年月/年季/年月日`，主图、KPI、达成率、同比均用此口径。
- **激励方案口径**（预留）：以 `承保时间` 为准，承保后 30 天内入账、再 15 天回销、未发生负金额。仅作字段入库，逻辑未实现。
- **基准年自动滚动**：见 §4.3。

### 5.2 维度归一化

#### 长短险

源数据可能出现 `长期 / 短期 / 极短期` 三值；入库时 **极短期合并入短期**，统一为 `长期 / 短期` 二值。

#### 外勤管理职白名单

`mgmt_level` 派生列，依据 `rank_level` 判定：

- **外勤管理职**：主管 / 经理 / 总监系列（按公司职级表枚举）
- **一线营销员**：客户经理及其他销售岗位

仅在 `fact_headcount` 入库阶段计算并落表，避免查询时反复 CASE。

#### 司龄分档

`tenure_band` 六档：a / b / c / d / e / f（具体月数边界以人力数据规范为准）。

### 5.3 列名兼容（COL_ALIASES）

源 Excel 列名存在多种写法，导入时通过别名表统一映射：

| 标准列 | 兼容写法 |
|--------|----------|
| 月 | `年月` |
| 季 | `年季` |
| 日期 | `年月日`、`投保日期`、`签单日期` |

缺失列会显示具体列名并给出最相近的实际列名建议（Levenshtein 模糊匹配）。

### 5.4 三个待确认的口径（重要）

详见 [指标口径手册.md §10](./docs/指标口径手册.md#10-待确认口径)：

| 编号 | 项目 | 占比 | 金额影响 | 当前处理 | 待确认 |
|------|------|------|----------|----------|--------|
| Q1 | **自保件**（self_mark='Y'） | 1.57% | ¥486 万 | 计入 | 是否剔除？ |
| Q2 | **空工号**（staff_id 缺失） | 19.54% | ¥2859 万 | 计入 | 是否剔除？归属？ |
| Q3 | **负金额**（撤单/退保） | 占比小 | -¥1.77 亿 | 计入 | 是否单独剥离？ |

这三项口径将影响所有保费类指标，**用户决策后需统一更新口径手册并回归全部模块**。

### 5.5 增量导入

- **保费数据**：完整覆盖（每次上传清空 `fact_premium` 重新建表）。
- **人力数据**（规划）：按 `period_ym` 增量（`DELETE WHERE period_ym=? + INSERT`），保留历史月份。
- **目标数据**：按 `year` 增量（`DELETE WHERE year=? + INSERT`），保留其他年度。

### 5.6 小计行过滤

源 Excel 常含汇总行（机构=「合计」、产品=「小计」），导入时按关键字黑名单过滤，避免重复计算。

### 5.7 buildWhere 模式

所有查询通过统一的 `buildWhere(filters)` 函数构建 WHERE 子句：

```js
const { where, params } = buildWhere({
  org: '北京',
  biz_mode: 'OTO',
  term_type: '长期',
  // ...
});
db.exec(`SELECT ... FROM fact_premium WHERE ${where}`, params);
```

避免字符串拼接，所有参数走 prepared statement。

## 7. 数据导入

### 7.1 保费 Excel 模板

**必需列**：

- **时间**：`年`、`月`（或 `年月`）、`季`（或 `年季`）、`日期`（或 `投保日期`/`签单日期`/`年月日`）
- **维度**：`销售机构名称`、`业务模式`、`长短险`、`是否商保年金产品`、`缴费年限`、`保障年限`、`产品设计分类`
- **指标**：`期交保费`、`年化规保`、`折算保费`（单位元）

**可选列**：`人员工号`、`主管工号`、`经理工号`、`投保单号`、`自保件标记`、`投保时间`、`承保时间`、`入账时间`、`回销时间`、`产品代码`、`产品名称`、`是否个人养老金`、`价值规保`

**解析流程**：
1. SheetJS 读取首个 Sheet → JSON
2. 列名模糊匹配（Levenshtein） → 报告缺失列
3. 行过滤（小计 / 合计行）
4. 构建 SQLite 表 → 批量 INSERT（金额 × 100 转分）
5. 写入 IndexedDB

### 7.2 目标 Excel 模板

**必需列**：

- **`年`**（必需）
- `季`、`月`、`周`（按时间维度填写，至少一项）
- `机构`、`项目`（可选，按维度填写）
- `折算保费目标`、`期交保费目标`、`规模保费目标`（至少一个）

**示例**：

| 年 | 季 | 月 | 机构 | 项目 | 折算保费目标 | 期交保费目标 |
|----|----|----|------|------|-------------|-------------|
| 2026 | Q1 | | 北京 | OTO | 50000000 | 80000000 |
| 2026 | Q1 | 1月 | 北京 | OTO | 15000000 | 25000000 |

**解析逻辑**：
1. 列名模糊匹配（年 / 季 / 月 / 周 / 机构 / 项目 / *目标）
2. 多指标列拆解为多行（每个指标列 → 一条 `metric, target_value` 记录）
3. `DELETE WHERE year=? + INSERT`（覆盖年度）
4. 写入 IndexedDB

### 7.3 人力 Excel 模板（规划）

详见 [人力数据规范.md](./docs/数据规范/人力数据规范.md)。核心列：

- `统计日期`、`销售机构名称`、`销售序列`、`职等`、`职级`、`人员代码`
- `月末司龄区间`、`业务模式名称`
- `月初在职人力`、`月末在职人力`（0/1）

**入库时派生**：`mgmt_level`（外勤管理职 / 一线营销员）。

### 7.4 错误处理

| 场景 | 处理 |
|------|------|
| 缺失必需列 | 弹错误提示，列出缺失列 + 推荐相近列名 |
| 必需列全空 | 跳过该行，最后汇总跳过数 |
| 金额非数字 | 视作 0，不报错 |
| 日期解析失败 | 走 fallback 链：`日期` → `投保日期` → `签单日期` → `年-月-日` 拼接 → 跳过 |
| 文件超大（>50MB） | 提示等待，浏览器 tab 可能短暂卡顿 |

## 8. 已知限制与待办

### 8.1 数据限制

1. **队伍人力数据近似**：当前基于 `fact_premium` 出单记录推算在职人力，无独立人力档案。
   - 月末在职 = 该期间有出单的去重工号数
   - 月均在职 = (上期期末 + 本期期末) / 2
   - 在网电多元渠道下相对合理（不出单≈无收入），但精度不及真实人力快照。
   - **解决路径**：完成 `fact_headcount` 表导入流程（已在数据规范中定义）。

2. **三个待确认口径**（自保件 / 空工号 / 负金额）：见 §5.4。

3. **目标数据格式**：当前需手动准备 Excel，模板下载未实现。

### 8.2 技术限制

4. **ESM 与内联不一致**：5 个新模块（modal / target-importer / kpi / team-performance / 部分 platform-trend 弹窗）的 ESM 文件均为占位，实际逻辑在 `经营分析模板.html` 内联 IIFE。`build.sh` 生成的 dist 版本依赖内联实现，ESM 版本（HTTP 服务模式）功能不全。

5. **趋势弹窗图表重渲染**：弹窗关闭再打开时，ECharts 实例需重新 `init` 而非 `setOption`，存在短暂闪屏。

6. **大文件性能**：>10万行 Excel 解析 5-30 秒，期间页面无响应（同步操作）。

### 8.3 待办清单

#### 高优先级

# v1.0.66 发布说明

发布日期：2026-05-26

## 一、发布结论

本版本修复权限管理模块两个问题：管理员可维护管理员组成员，权限管理弹窗中的“保存”按钮固定到独立操作列。同步增加后端防锁死保护，避免误操作导致系统没有可用管理员。同时将“队伍结构与产能分析”从试运行调整为正式模块，并修复登录态下该模块接口未携带鉴权信息导致无数据的问题。

## 二、主要变更

1. 管理员可新增管理员账号。
2. 管理员可将普通用户组、高级用户组账号提升为管理员组。
3. 管理员可将其他管理员组账号降级为高级用户组或普通用户组。
4. 禁止降级或停用当前登录管理员。
5. 禁止移除系统中最后一个可用管理员账号。
6. 权限管理表格固定列宽，“保存”按钮固定在最右侧独立操作列，避免与模块权限复选框混在一起。
7. “队伍结构与产能分析”去除试运行标识，新增独立权限项，默认管理员组、高级用户组、普通用户组均可查看。
8. 前端改用统一鉴权请求读取 `/api/team-enhanced-analysis`，避免登录后接口 401 导致页面显示“暂无数据”。

## 三、验证口径

- 后端权限回归测试覆盖管理员提升、降级、自我降级拦截和最后管理员保护。
- 前端静态测试覆盖管理员组角色选项、保存按钮布局、队伍结构与产能分析正式模块权限和鉴权请求。
- 前端缓存标识更新至 `v1.0.66`，避免浏览器继续加载旧脚本。

## 四、注意事项

管理员组账号默认拥有全部模块权限。若需要调整某个管理员账号的模块权限，应先将其降级为高级用户组或普通用户组，再进行细项权限配置。

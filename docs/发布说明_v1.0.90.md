# v1.0.90 发布说明：荣誉体系人员历史与管理职团队追踪

## 一、更新结论

本版本在“荣誉体系 / 星钻联盟”看板中补充两类关键追踪视角：专员历史月度明细、主管/经理历史团队表现。新增内容不改变主看板 KPI、机构维度、队伍结构与产能分析的既有计算链路。

## 二、主要变化

1. 新增专员历史月度明细：支持按机构、项目、人员代码、人员姓名、历史年月查看期交保费、标保、长险件数、月度获扣钻、累计钻石和会员等级。
2. 新增管理职历史团队表现：支持按机构、项目、主管/经理代码、主管/经理姓名、历史年月查看团队会员人数、团队期交、团队标保、团队钻石和月度获扣钻情况。
3. 前端在“专员级”和“管理职”页内增加历史明细区和筛选条件，避免另设入口造成使用路径分散。
4. 后端 `/api/honor/dashboard` 增加 `specialistHistory` 和 `managerHistory` 数据段，统一由服务端输出，前端只负责筛选与展示。

## 三、计算与归集口径

### 1. 专员历史

- 人员范围：`honor_person_month` 中非主管、非经理人员。
- 业绩来源：按人员代码、项目、月份匹配 `honor_source_policy`。
- 展示指标：期交保费、标保、长险件数、月度达标、月度钻石变化、累计钻石、会员等级、新星人力标识。

### 2. 管理职历史

- 管理职范围：`honor_source_staff_month` 中角色为主管或经理的人员。
- 主管团队：按营业组编码归集。
- 经理团队：按营业部编码归集。
- 团队会员人数：团队成员中会员等级不为“未入会”的人数。
- 团队业绩：团队成员对应月份的期交保费与标保合计。
- 缺团队编码：不伪造团队数据，展示“缺团队编码，暂不归集团队”。

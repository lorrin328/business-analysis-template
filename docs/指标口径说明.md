# 指标口径说明

## 核心原则

所有核心指标公式统一放在 `backend/metrics/`，前端不得重复编写经营指标公式。所有除法统一使用 `safe_divide`，自动处理 0、`None`、`NaN`、`Infinity`。

## 指标口径

| 指标 | 公式 | 单位 | 不可计算规则 |
|---|---|---|---|
| 达成率 | 实绩 / 目标 | % | 目标为空或为 0 |
| 同比 | 本期 / 去年同期 - 1 | % | 去年同期为 0 或缺失 |
| 环比 | 本期 / 上期 - 1 | % | 上期为 0 或缺失 |
| 序时进度 | 已过时间 / 总周期 | % | 周期参数缺失 |
| 进度偏差 | 达成率 - 序时进度 | 百分点 | 达成率或序时进度不可计算 |
| 活动率 | 活动人力 / 在职人力 | % | 在职人力为 0 或缺失 |
| 人均保费 | 月均新单保费 / 月均在职人力 | 万元/人/月 | 月均在职人力为 0 或缺失 |
| 人均产能 | 新单保费 / 月均活动人力 | 万元/人 | 月均活动人力为 0 或缺失 |
| 转化率 | 成交数 / 触达或线索数 | % | 基数为 0 或缺失 |
| 费用率 | 费用 / 保费 | % | 保费为 0 或缺失 |
| 投产比 | 产出 / 投入 | 倍 | 投入为 0 或缺失 |

## 去年同期定义

**去年同期指相对于当前统计日，上一年的同一日。**

- 例：当前数据截至 2026-05-21，则去年同期为 **2025-01-01 至 2025-05-21** 的累计数据。
- 不得用「上年 1-5 月全月累计」替代。

### 数据精度说明

| 指标类别 | 支持精度 | 说明 |
|---------|---------|------|
| 期交保费（转型/经代） | 日级 | 使用 `agg_daily_performance` / `agg_jingdai_daily` 日累计表；转型与经代日表截止日不一致时，统一取双方共同最早截止日 |
| 机构维度期交保费 | 日级 | 使用 `agg_org_daily_performance`，已实现同口径截止 |
| 人力指标（活动率、人均保费、人均产能） | 月级 | 人力基表 `agg_hr_data` 无日维度，按「截至同月」计算；人均保费在任意日期范围下按覆盖自然月数和覆盖月份月均在职人力折算 |
| 价值保费 | 月级 | `agg_value_data` 无日维度，按「截至同月」计算 |
| 长险期交 | 月级 | `agg_longterm_qj` 无日维度，按「截至同月」计算 |
| 产品结构（年金/10年/保障） | 月级 | `agg_org_performance` 无日维度，按「截至同月」计算 |

## API 返回要求

统一 API 返回中需通过 `meta.metric`、`meta.unit`、`meta.dataSource` 标明指标、单位和数据来源；复杂指标在 `meta.definitions` 中返回口径说明。

## 展示约束

1. 长险活动率同比按百分点差展示，即“本期活动率 - 上年同期活动率”，单位为 `pp`，不使用相对增速。
2. 长险期交、保障类产品等聚合表暂无完整数据支撑时，前端展示为空状态或“口径待完善”，不得默认展示 `0%` 作为达成结果。
3. 服务端目标返回 `categories: null` 时，前端可使用默认目标保证页面可用，但必须标明“服务端尚未配置正式目标”。
4. 期交保费、平台日累计趋势等日级指标必须保证转型与经代按同一截止日计算；当前执行规则为取共同最早截止日，后续导入遇到转型/经代日级截止日不一致时拒绝写入。
5. 长险期交达成率分母沿用期交保费目标，不另设独立长险期交目标。

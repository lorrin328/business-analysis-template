# 数据流说明

## 当前数据流

1. 用户上传业绩、经代、人力、价值 Excel。
2. `backend/aggregator.py` 解析并聚合为服务端表。
3. `backend/services/data_transform.py` 统一月份、数字、机构、业务线等字段转换规则。
4. `backend/validators/` 对导入结果做基础校验。
5. 聚合结果写入 `business_data.db`。
6. 前端通过 FastAPI 接口读取标准数据并渲染图表。

## 关键表

| 表 | 用途 |
|---|---|
| `agg_performance` | OTO、证保、蚁桥月度业绩 |
| `agg_jingdai` | 经代月度业绩 |
| `agg_daily_performance` | 转型业务日累计 |
| `agg_jingdai_daily` | 经代日累计补充数据 |
| `agg_org_performance` | 机构维度业绩 |
| `agg_hr_data` | 人力、活动人力 |
| `agg_value_data` | 价值保费 |
| `target_config` | 目标配置 JSON，兼容当前页面 |
| `target_values` | 目标行表，支持年度、季度、月度、业务线、机构和指标查询 |

## 数据校验

严重错误阻断导入：必填字段缺失、月份异常、负数金额。一般警告允许导入但需记录：机构未登记、业务线未登记、重复键。

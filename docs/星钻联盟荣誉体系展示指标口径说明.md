# 星钻联盟荣誉体系展示指标口径说明

## 一、总体结论

星钻联盟荣誉体系当前展示口径聚焦 OTO、证保两类业务，不纳入经代、蚁桥（网服）。页面展示的核心目标不是替代经营看板，而是围绕“谁在追踪、谁已入会、谁在获钻、谁在扣减、哪些机构和团队需要督导”形成专项管理视图。

当前展示口径遵循四项原则：

1. 只统计 OTO、证保人员及业绩，经代、蚁桥（网服）不计入个人星钻和团队星钻。
2. “追踪人力”采用最新人力清单当前在职口径；历史已离职人员只进入累计追踪池，不计入当前有效追踪人力。
3. 最新人力清单中未出现，或最新人力清单显示月末在职人力为 0，均按离职或非在职处理，当前钻石清零。
4. 页面奖励金额均为测算值，仅用于管理预估，不代表最终发放金额。

## 二、适用范围与基础口径

| 项目 | 口径说明 |
|---|---|
| 适用业务 | OTO、证保 |
| 不适用业务 | 经代、蚁桥（网服） |
| 统计周期 | 以页面选择的年份、月份为准；当前页面默认展示 2026 年 5 月 |
| 人员范围 | 当年截至统计月，在人力基表中出现过的 OTO、证保人员 |
| 当月有效人力 | 最新人力清单存在，且月末在职人力大于 0 的人员 |
| 累计追踪池 | 当年截至统计月曾进入星钻追踪范围的人员，包含历史已离职或当月非在职人员 |
| 离职或非在职清零 | 最新人力清单不存在，或最新人力清单显示月末在职人力为 0，当前钻石清零，会员等级显示为未入会 |
| 业绩归属 | 按承保时间归属月份；承保时间缺失时按入账时间或年月字段降级归属，并纳入异常提示 |
| 回销跟踪 | 保单按承保时间先计入星钻统计；承保满 45 个自然日后仍未回销成功，或后续发生犹豫期退保、负保费冲减时，再不计入星钻统计 |
| 长险件数 | 取长短险字段中符合“一年期以上”的保单件数 |
| 保费口径 | 以折算保费作为月度达标判断口径；期交保费、年化规保用于明细展示和辅助核对 |

## 三、星钻计算基础规则

| 规则 | OTO | 证保 |
|---|---:|---:|
| 月度达标门槛 | 折算保费不低于 20,000 元，且长险不少于 1 件 | 折算保费不低于 30,000 元，且长险不少于 1 件 |
| 达标结果 | 获 1 颗钻石 | 获 1 颗钻石 |
| 未达标处理 | 非保号月扣 1 颗，最低扣至 0 | 有长险但未达标为保号月，不扣减；无长险且未达标扣 1 颗，最低扣至 0 |
| 非在职处理 | 当月清零 | 当月清零 |

会员等级按当前累计钻石匹配最高达成等级。低于 3 颗为未入会；达到 3 颗起进入初级会员，并随累计钻石提升到中级、高级、资深、黄金、白金、钻石、至尊、金星、恒星、星钻、星曜等级。

新星人力指当年入职且入职后 3 个月内累计钻石达到 3 颗及以上的人员。

## 四、顶部指标卡片

| 展示指标 | 指标含义 | 取数口径 | 计算逻辑 | 管理用途 |
|---|---|---|---|---|
| 追踪人力 | 当前仍在星钻追踪范围内的有效人员 | 最新人力清单中 OTO、证保人员，且月末在职人力大于 0 | 按机构、项目汇总当前有效人员数 | 判断当前可督导、可转化的人力基盘 |
| 累计追踪池 | 当年曾进入星钻追踪范围的全部人员 | 当年截至统计月出现过的 OTO、证保人员 | 按人员代码和业务线去重统计，包含历史已离职人员 | 区分“历史池子”和“当月有效人力”，避免误读 |
| 离职/非在职清零 | 当前已不计入有效追踪的人数 | 最新人力清单不存在，或月末在职人力为 0 的人员 | 统计被标记为非在职并清零的人员数 | 识别人员流失对星钻沉淀的影响 |
| 会员人数 | 当月有效追踪人力中已入会的人数 | 当月有效人员，且会员等级不为未入会 | 按人员数汇总 | 衡量星钻体系实际沉淀人群 |
| 会员率 | 当月有效追踪人力中的入会占比 | 会员人数、追踪人力 | 会员人数 ÷ 追踪人力 | 判断机构和项目的荣誉转化效率 |
| 资深及以上 | 当月有效人员中达到资深会员及以上的人数 | 会员等级达到资深、黄金、白金、钻石、至尊、金星、恒星、星钻、星曜 | 按人员数汇总 | 识别重点荣誉人群和高价值维护对象 |
| 本月获钻 | 统计月钻石增加的人员数 | 当月钻石变化大于 0 的人员 | 按人员数汇总 | 判断当月达标和激励产出情况 |
| 本月扣减 | 统计月钻石减少的人员数 | 当月钻石变化小于 0 的人员 | 按人员数汇总 | 锁定需要优先督导和恢复达标的人员 |
| 累计钻石 | 当前批次全部人员的钻石余额合计 | 当年截至统计月的人员钻石余额 | 按人员当前钻石余额求和 | 判断荣誉资产沉淀规模 |
| 新星人力 | 新人中已达到新星条件的人数 | 当年入职、入职 3 个月内、累计钻石达到 3 颗及以上 | 按人员数汇总 | 衡量新人培育和早期荣誉转化 |
| 测算奖励 | 按会员等级估算的奖励金额 | 当月有效人员的会员等级 | 初级至资深 100 元，黄金至至尊 200 元，金星及以上 300 元 | 用于奖励预算预估，不作为最终发放 |
| 异常数量 | 影响口径核对或结果解释的数据问题数 | 字段缺失、日期降级、负保费、人员工号缺失等异常 | 按异常记录条数汇总 | 提醒先处理数据问题再做结果判断 |

## 五、总览驾驶舱

总览驾驶舱用于快速形成经营判断，主要展示四类结论：

| 展示内容 | 口径说明 | 计算逻辑 | 管理用途 |
|---|---|---|---|
| 当前会员率 | 当月有效追踪人力中的会员占比 | 会员人数 ÷ 追踪人力 | 判断星钻体系总体转化水平 |
| 机构领先 | 会员率和累计钻石表现靠前的机构及项目 | 按机构、项目排序，优先看会员率，其次看累计钻石 | 识别可复制经验 |
| 扣减关注 | 本月扣减人数较高的机构及项目 | 按本月扣减人数倒序识别 | 确定督导优先级 |
| 月度预警 | 当前需要关注的人员或数据事项数量 | 汇总扣减、保号、未达标、异常数据等预警 | 形成后续追踪清单 |

总览中的“项目表现”按 OTO、证保展示追踪人力、会员人数、会员率、本月获钻和本月扣减；“各机构会员结构”展示各机构会员人数及其中专员级、管理职拆分；“月度降级预警”展示本月较上月发生等级下降的人员、简要原因和标保差额。

## 六、机构追踪

机构追踪按“机构 + 项目”展示各机构星钻经营结果。

| 展示指标 | 指标含义 | 取数口径 | 计算逻辑 |
|---|---|---|---|
| 排名 | 当前筛选范围内的机构排序 | 机构、项目汇总结果 | 优先按会员率排序，其次按累计钻石排序 |
| 机构 | 人员所属销售机构 | 人力基表中的销售机构名称 | 按人员最新归属展示 |
| 项目 | OTO 或证保 | 人力基表和业绩明细中的业务模式 | 仅保留 OTO、证保 |
| 追踪人力 | 机构项目下当月有效人员数 | 当月在职人员 | 按人员数汇总 |
| 会员人数 | 机构项目下当月有效会员数 | 当月有效人员，且会员等级不为未入会 | 按人员数汇总 |
| 会员率 | 机构项目下会员转化比例 | 会员人数、追踪人力 | 会员人数 ÷ 追踪人力 |
| 人均钻石 | 机构项目下人均钻石余额 | 累计钻石、追踪人力 | 累计钻石 ÷ 追踪人力 |
| 本月获钻 | 机构项目下当月获钻人数 | 当月钻石变化大于 0 的人员 | 按人员数汇总 |
| 本月扣减 | 机构项目下当月扣减人数 | 当月钻石变化小于 0 的人员 | 按人员数汇总 |
| 累计钻石 | 机构项目下人员当前钻石余额合计 | 人员当前钻石余额 | 按钻石数求和 |
| 测算奖励 | 机构项目下预计奖励合计 | 当月有效人员会员等级 | 按等级对应金额求和 |

该页签主要用于横向比较机构表现，建议重点看会员率、本月扣减和人均钻石三个指标。

## 十、月度预警

月度预警用于生成需要优先处理的人员清单和数据清单。

| 预警类型 | 触发条件 | 展示字段 | 建议处理方向 |
|---|---|---|---|
| 月度扣减 | 当月钻石变化小于 0 | 月份、机构、项目、人员、层级、会员等级、当前钻石、本月变化、标保、长险件数 | 督导次月恢复达标，重点关注连续扣减人员 |
| 等级下降 | 本月会员等级低于上月会员等级 | 上月等级、本月等级、标保、标保差额、长险件数、简要原因 | 优先督导恢复达标或确认离职清零 |
| 数据异常 | 出现字段缺失、日期降级、负保费、人员工号缺失等问题 | 机构、人员、异常类型、建议动作 | 先复核源数据，再判断经营结果 |

预警优先级按“数据错误或严重异常、等级下降”排序。月度经营督导建议优先处理等级下降人员，并区分“离职/非在职清零、缺少长险件、标保不足”等原因。

## 十一、人员明细

人员明细用于按个人查看当前星钻结果。

| 展示指标 | 指标含义 | 计算逻辑 |
|---|---|---|
| 机构 | 人员所属机构 | 取人员最新归属机构 |
| 项目 | OTO 或证保 | 取人员业务模式 |
| 层级 | 个人、主管、经理 | 按职等、职级识别 |
| 人员代码 | 人员唯一识别编码 | 取人力基表人员代码 |
| 人员姓名 | 人员姓名 | 取人力基表人员姓名 |
| 会员等级 | 当前星钻等级 | 按当前钻石余额匹配等级门槛；非在职显示未入会 |
| 当前钻石 | 当前累计钻石余额 | 按月度获钻、扣减、清零滚动计算 |
| 累计获钻 | 当年累计获得的钻石数 | 累加当年月度正向钻石变化 |
| 累计扣减 | 当年累计扣减的钻石数 | 累加当年月度负向钻石变化的绝对值 |
| 达标月份 | 当年月度达标次数 | 月度达标为 1 的月份数 |
| 新星人力 | 是否符合新星条件 | 当年入职、入职 3 个月内、累计钻石达到 3 颗及以上 |

该页签适合用于个体复盘和机构沟通，重点核对“达标月份、累计获钻、累计扣减、当前钻石、会员等级”。

## 十二、数据审计

数据审计用于判断现有数据是否足以支撑星钻计算，不作为经营结果排名。

| 展示内容 | 口径说明 |
|---|---|
| 是否可复用现有数据 | 判断现有业绩明细、人力基表是否覆盖星钻计算所需字段 |
| 必需字段覆盖 | 统计月度计算必须字段的覆盖数量，包括统计年月、机构、业务模式、人员代码、入职年月、月末在职、折算保费、长短险、承保时间等 |
| 可选字段覆盖 | 统计辅助展示字段的覆盖数量，包括人员姓名、职等职级、期交保费、年化规保、团队代码、产品字段等 |
| 是否新增上传 | 当前口径为不新增星钻专用上传，优先复用现有数据 |
| 字段明细 | 展示字段名称、匹配字段、是否必需、是否可用、缺失影响和降级处理方式 |

数据审计的管理含义是：先确认数据能否支撑，再判断星钻结果是否可用于督导。

## 十三、导出 Excel

导出文件用于离线复核和材料留痕，包含以下内容：

| Sheet | 内容 | 用途 |
|---|---|---|
| 总览 | 顶部核心指标汇总 | 快速复核总量结果 |
| 机构汇总 | 机构、项目层面的追踪人力、会员、获钻、扣减、奖励测算 | 机构横向对比 |
| 人员汇总 | 人员当前钻石、会员等级、累计获钻、累计扣减、达标月份、新星标识 | 个人结果核对 |
| 月度明细 | 人员逐月标保、长险件数、达标、保号、获扣钻、余额 | 过程追溯 |
| 季度奖励测算 | 按会员等级形成的预计奖励 | 奖励预算测算 |
| 异常清单 | 数据异常和建议处理动作 | 数据修正和口径解释 |
| 字段审计 | 字段覆盖情况 | 数据适配留痕 |
| 规则口径 | 会员等级、月度达标、保号、清零、奖励等规则说明 | 口径说明 |

导出结果与页面使用同一批次数据。若页面重新计算后再导出，导出文件反映重新计算后的结果。

## 十四、常见误读提醒

1. “追踪人力”不是历史累计人数，而是最新人力清单中的当前在职人员。
2. “累计追踪池”包含历史离职人员，不能直接作为当前可督导人力。
3. “离职/非在职清零”是荣誉体系计算规则，不等同于经营看板的人力流失分析。
4. “本月获钻”统计的是人员数，不是钻石颗数；当前规则下单人单月最多获 1 颗。
5. “测算奖励”只用于预算预估，不代表最终发放。
6. “团队表现”只做管理观察，不改变个人星钻结果。
7. 经代、蚁桥（网服）不纳入星钻计算，不应被解释为数据缺失或计算遗漏。

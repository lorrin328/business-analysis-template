# v1.0.48 发布说明

## 版本信息

- 版本号：v1.0.48
- 发布日期：2026-05-25
- 类型：fix / data-quality

## 问题背景

参数设置中出现 `4281` 与 `4281.0` 两条产品记录。根因是 Excel 读取产品代码时可能把数字型代码转成浮点样式，而参数配置自动抽取、保存与聚合计算没有在所有链路上统一执行产品代码规范化。

## 修复内容

1. 产品参数自动抽取时统一规范产品代码，将 `4281.0` 规范为 `4281`。
2. 参数保存时统一规范产品代码，避免用户保存 `4281.0` 后新建重复记录。
3. 读取 `product_config` 进行商保年金、保障类产品聚合时同步规范产品代码，避免配置与原始保单代码不一致导致漏算。
4. 新增历史重复配置合并逻辑，同一业务类型下 `4281` 与 `4281.0` 自动合并。
5. 合并规则采用“Y 优先”：任一重复记录标记为商保年金或保障类，则合并后保留 Y，避免用户配置丢失。

## 风险判断

- 重复记录本身会造成参数设置页面冗余。
- 若 `4281` 与 `4281.0` 两条记录配置不一致，旧逻辑存在漏算或错算风险。
- 本版本已在抽取、保存、读取配置、历史清理四个环节统一口径，避免重复再次产生。

## 验证结果

- 新增测试覆盖：
  - `4281` 与 `4281.0` 自动合并为一条配置。
  - 保存 `4281.0` 时实际更新 `4281`。
  - 聚合计算能用旧配置 `4281.0` 命中原始保单 `4281`。
- 本地回归测试：`167 passed`

## 部署提示

部署本版本后，打开“参数设置”或保存产品参数时会自动清理历史重复产品代码。建议部署后进入参数设置页面核对一次产品清单。

# v1.0.75 发布说明

## 发布类型

fix / import / data-refresh

## 核心结论

本版本修复“导入 Excel 后 KPI 概览、机构维度等看板数据没有变化”的主要风险点。此前系统会按文件哈希跳过历史已成功导入的重复文件，页面提示又容易让用户误以为已重新写库。本版本将页面手工导入调整为强制重新解析和写入，确保用户主动上传 Excel 后，原始表和聚合表都会刷新。

## 主要变更

1. 后端 `/api/upload`、`/api/import` 新增 `force` 参数。
2. 前端上传 Excel 默认调用 `/api/upload?force=true`。
3. 重复文件在 `force=true` 场景下不再跳过，会重新写入相关原始表和聚合表。
4. 上传结果提示明确区分“已重新写入并刷新看板数据”和“重复文件未写入”。
5. 补充强制导入相关测试，避免后续恢复为静默跳过。

## 验证重点

- 选择 Excel 上传后，接口请求包含 `force=true`。
- 重复文件在 Web 手工导入场景下仍会进入解析与聚合写入链路。
- 导入完成后页面执行刷新逻辑，重新读取 KPI、机构维度、交期结构和数据截止。
- 自动化测试通过。

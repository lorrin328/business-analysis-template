# 部署说明

## 推荐部署

Ubuntu Server + Nginx + systemd + FastAPI + SQLite。

## 一键部署

```bash
sudo bash deploy/deploy.sh
```

部署前建议先执行上线前检查：

```bash
bash deploy/check_before_deploy.sh
```

Windows 本地检查可执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\preflight.ps1
```

> 手动/systemd 部署要求服务器 `python3` 版本为 3.10 或更高。旧系统 Python 不满足要求时，请先升级 Python；本项目当前不再维护 Docker 部署链路。

脚本会完成：

1. 安装 Python、Nginx、rsync。
2. 同步项目到 `/opt/business-analysis`。
3. 创建 Python 虚拟环境并安装依赖。
4. 初始化 SQLite 表和索引。
5. 安装 systemd 服务。
6. 安装 Nginx 配置并重启服务。

上线前检查会确认核心文件存在、Nginx 上传限制已配置、自动测试通过、数据质量审计无异常。

## 当前部署边界

- 生产部署只保留 `Nginx + systemd + FastAPI + SQLite`。
- Dockerfile、docker-compose、Docker 专用 Nginx 配置和 Docker 镜像构建工作流已归档到 `bak/20260524_stability_archive/deprecated_docker/`。
- 本地虚拟环境、缓存、旧构建产物和历史数据库备份已归档到 `bak/20260524_stability_archive/`，不参与部署同步。
- Excel 源文件保留在项目根目录或 `excel/` 目录，部署脚本仍按根目录 Excel 自动重建数据库。

## 常用命令

```bash
sudo systemctl status business-analysis
sudo journalctl -u business-analysis -f
sudo systemctl restart business-analysis
tail -f /opt/business-analysis/backend/logs/app.log
curl http://127.0.0.1:45679/api/health
```

## 数据备份

部署脚本会在覆盖前备份 `/opt/business-analysis/backend/business_data.db` 到 `/opt/business-analysis-backups`。

## 端口

后端服务固定监听 `0.0.0.0:45679`，Nginx 负责对外代理访问。

# v1.0.39 发布说明

## 核心修复

修复 KPI 概览“人均保费”明显偏低的问题。

## 问题原因

当时修复的是分母重复放大问题；后续版本已进一步明确人均保费采用“月均新单保费 / 月均在职人力”。

```text
月均新单保费 / 月均在职人力
```

此前前端在 KPI 卡片中使用了后端返回的 `avg_sum`，但没有除以 `months`。因此 1-5 月展示时，分母被放大约 5 倍，导致人均保费明显偏低。

## 修复内容

1. KPI 概览卡片
   - 分母改为各渠道 `avg_sum / months` 后再汇总。
   - 当前本地数据测算：错误口径约 `6.5万`，修复后约 `32.3万`。

2. 人均保费弹窗
   - 年度累计、季度累计均改为对应期间月均新单保费除以对应期间月均在职人力。
   - 当月口径保持当月保费除以当月月均在职人力。

3. 人均产能检查
   - 当前趋势图为月度口径，使用当月保费除以当月活动人力，不存在同类累计分母放大问题。

4. 回归测试
   - 新增前端静态测试，防止再次使用多个月人力累计作为人均类分母。

## 验证

- 轻量测试：`105 passed, 8 skipped`
- 完整 FastAPI 测试：`147 passed`

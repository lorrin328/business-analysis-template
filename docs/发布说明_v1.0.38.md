# v1.0.38 发布说明

## 核心目标

本版本面向后续综合性经营看板扩展，重点增强数据定义、聚合重建、导入安全和部署复现能力，降低因 Excel 更新节奏、SQLite 聚合滞后、生产无源文件导致的数据口径风险。

## 主要变更

1. 新增 SQLite 原始表聚合重建能力
   - 新增 `backend/services/aggregate_rebuilder.py`
   - 新增 `backend/rebuild_aggregates_from_raw_tables.py`
   - 生产环境即使没有 Excel，也可以从 `performance`、`jingdai`、`hr_data`、`value_data` 原始表重建所有 `agg_*` 聚合表。

2. 统一日级截止日策略
   - 新增 `backend/services/cutoff_policy.py`
   - KPI 继续按转型、经代各自真实截止日计算。
   - 同时保留 `common` 共同截止日，供趋势线、混合对比等需要同日口径的场景使用。

3. 提升导入安全
   - Excel 上传默认不再允许部分失败后写入成功文件。
   - 避免生产看板出现“部分新数据 + 部分旧数据”的混合状态。
   - 如确需部分导入，可显式传入 `allow_partial=true`。

4. 部署增强
   - `deploy/deploy.sh` 在没有 Excel 时会自动尝试从 SQLite 原始表重建聚合。
   - Webhook 缺少 `WEBHOOK_SECRET` 时拒绝启动，避免无签名部署入口。

5. 管理端安全调整
   - 管理员 Token 前端缓存从 `localStorage` 改为 `sessionStorage`。
   - 文档移除默认明文 Token 示例，生产环境必须显式配置 `ADMIN_TOKEN`。

## 验证结果

- 轻量测试：`104 passed, 8 skipped`
- 完整 FastAPI 测试：`146 passed`
- 本地 SQLite 原始表重建验证成功：
  - 原始表：`performance=70221`、`jingdai=131661`、`hr_data=16234`、`value_data=153`
  - 重建年份：`2022-2026`
  - 重建聚合表包含：`agg_daily_performance`、`agg_jingdai_daily`、`agg_org_performance`、`agg_payment_period`、`agg_longterm_qj` 等核心表。

## 部署注意

生产部署后建议检查：

1. 页面版本号是否为 `v1.0.38`
2. `/api/health` 是否返回正常
3. `/api/kpi/2026` 中 `daily_cutoff.transform`、`daily_cutoff.jingdai` 是否分别体现真实数据截止日
4. KPI 概览、机构分析、业务平台趋势、产品结构、交期结构是否均正常出图
5. 上传 Excel 时，如有任一文件失败，系统应拒绝写库并提示重新上传

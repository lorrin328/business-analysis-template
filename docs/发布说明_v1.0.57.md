# v1.0.57 发布说明

## 结论

本版本完成两项口径和展示修复：一是价值达成率正式纳入经代口径，在经代价值数据表尚未接入前，经代实绩显式按 0 展示；二是期交保费达成率弹窗表格增加同比列，便于在明细层面核对整体、经代、转型和各转型子渠道的同比表现。

## 主要变化

1. `/api/kpi` 的 `value` 数据显式返回 `经代: 0`。
2. KPI 卡片、价值弹窗、Excel 导出口径说明均明确价值达成率包含经代。
3. 期交保费达成率弹窗年度、季度、月度表格新增“同比”列。
4. 年度同比优先使用服务端 `qj_premium_prev`，季度和月度同比使用上一年平台数据。
5. `kpi-cards.js`、`kpi-modal-content.js` 同步增加版本缓存标识。
6. 指标计算口径核对报告和最终审计口径报告已同步更新。

## 口径说明

```text
价值达成率 = （经代价值保费 + OTO价值保费 + 证保价值保费 + 蚁桥价值保费） / 价值整体目标
```

当前经代价值数据表尚未接入，因此经代价值实绩暂按 0 展示；后续接入经代价值数据后，整体价值达成率将自动纳入经代实绩。

```text
期交弹窗同比 = 本年对应口径累计 / 去年同期对应口径累计 - 1
```

## 验证

- 前端静态测试覆盖期交弹窗同比列、经代价值占位口径和脚本缓存标识。
- 本地全量回归测试：`175 passed`。

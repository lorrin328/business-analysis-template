# v1.0.91 发布说明：人员管理入口与基本法测算模块

## 一、发布结论

本版本新增“人员管理”独立入口，并将外部人管小程序中的 OTO、证保基本法测算能力迁移为现有 Web 看板可用的原生页面。新增入口默认仅管理员组可见，不引入微信云开发、云存储 JSON 和前端本地机构选择权限逻辑。

## 二、功能变化

1. 主看板右上角新增“人员管理”按钮，使用 `personnel_management` 权限控制。
2. 新增 `personnel-management.html` 独立页面，承接基本法测算模块。
3. OTO 测算支持人产、缴期、机构类型、产品系数，输出个人 NFYP、团队 NFYP、UM 团队 NFYP、固定收入、浮动收入、团队提奖、主管/经理利益分摊、成本合计和费用率。
4. 证保测算支持标准、费优、其他三类费用版本，输出 FYC、业务职级、基本佣金、绩效佣金、管理津贴、育成津贴、经理分摊基本佣金、成本合计和基本法费用率。

## 三、权限口径

- 管理员组：默认拥有人员管理权限。
- 高级用户组：默认不开放人员管理权限，可由管理员在权限管理中单独授权。
- 普通用户组：默认不开放人员管理权限。

## 四、技术说明

- 新增权限键：`personnel_management`。
- 新增前端页面：`personnel-management.html`。
- 新增前端脚本：`js/personnel-management.js`。
- 页面复用现有登录态，不新增微信云数据库、云存储或本地身份选择机制。

## 五、验证范围

- 权限默认值回归：管理员拥有全部权限，高级用户和普通用户默认不具备人员管理权限。
- 前端静态校验：确认主页面入口、独立页面脚本、OTO/证保测算函数和权限拦截存在。
- 版本号统一：主看板、荣誉体系、AI 只读快照与前端缓存标识同步到 v1.0.91。

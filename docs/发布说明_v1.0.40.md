# v1.0.40 发布说明

## 核心修复

继续修复人均保费偏低的生产数据根因：生产 SQLite 中 `hr_data` 原始人力明细存在重复追加，导致 `agg_hr_data` 月初/月末在职人力被放大。

## 问题原因

人力 Excel 的月份列可能是 `YYYY-MM-DD` 日期格式。此前增量导入删除旧月份数据时，只兼容 `202605` 等紧凑年月格式，未正确删除日期格式的旧数据。多次上传后，原始 `hr_data` 被重复追加，进而放大月均在职人力分母。

## 修复内容

1. 导入删除逻辑
   - `hr_data` 等含年列 + 月份列的原始表，现在会先去除 `-`、`/` 后再按 `YYYYMM` 匹配删除。

2. 聚合重建保护
   - 从 SQLite 原始表重建聚合时，对原始明细先执行去重，避免历史重复明细继续污染聚合。

3. 回归测试
   - 覆盖日期型月份列的增量替换。
   - 覆盖原始表重复行去重后再重建聚合。

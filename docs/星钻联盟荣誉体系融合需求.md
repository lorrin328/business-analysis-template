# 星钻联盟荣誉体系融合需求

本模块在当前经营分析平台内新增 `honor` 独立业务域，不新增 Node.js/Express 服务，不新增独立登录体系，不默认新增星钻专用 Excel 上传。

本期实施边界：

1. 优先复用现有 `performance`、`hr_data`、`jingdai`、`value_data`。
2. 先完成字段审计，再进入个人星钻 MVP。
3. 星钻仅覆盖转型 OTO、证保；经代和蚁桥（网服）均不涉及星钻。
4. 团队星钻、证保季度通算、规则版本管理、历史批次对账进入后续阶段；团队口径只归集 OTO/证保，团队内网服/蚁桥人员及业绩均排除。
5. 所有计算结果保留 `batch_id` 和 `rule_version`。
6. 字段不足、日期降级、负保费等问题进入 `honor_exceptions`。

详细字段适配结论见 `docs/honor_data_assessment.md`。

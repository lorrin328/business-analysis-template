## 1. 文件命名

```
N1AI-人力基表_<导出日期>_<导出时间序号>.xlsx
例：N1AI-人力基表_20260421_1637302.xlsx
```

- 文件名仅作识别
- 同一文件可包含多个统计月份（实测：单文件含 4 个月）
- 文件应在 `.gitignore` 中保护（含员工姓名等敏感信息）

## 3. 关键约束与校验

### 3.1 ⚠️ 小计行过滤（必做）

实测文件含 4 行小计行（每月 1 条），特征：

```
销售机构名称 = '小计'
销售序列 = '小计'
职等 = '小计'
月初在职人力 = 大数值（如 560）
```

**导入逻辑**：

```sql
-- 任何维度等于 '小计' 即跳过
WHERE org = '小计' OR sales_seq = '小计' OR rank_grade = '小计'
   OR rank_level = '小计' OR biz_mode = '小计' OR tenure_band = '小计'
```

或在 transformRow 中：

```js
if ([org, sales_seq, rank_grade, rank_level, biz_mode, tenure_band]
    .some(v => v === '小计')) return null;  // 跳过
```

如果不过滤，将导致**人力被重复计 2 倍**。这是数据正确性的关键。

### 3.2 唯一性约束

逻辑主键：`(period_ym, staff_id)` —— 每月每人一条。

入库前检查重复：

```sql
CREATE UNIQUE INDEX ux_period_staff ON fact_headcount(period_ym, staff_id);
```

如同一文件内或跨文件重复，按"后导入覆盖"处理（DELETE then INSERT）。

### 3.3 与保单表的关联校验

导入后建议做一次连贯性检查：

```sql
-- 保单表中存在但人力表中找不到的工号
SELECT DISTINCT staff_id FROM fact_premium
WHERE staff_id NOT IN (SELECT staff_id FROM fact_headcount)
  AND staff_id != '';
```

期望：少量（试用期、临时工等），数量过大说明人力数据缺失月份。

### 3.4 缺失月份提示

启动时检查人力表的 `period_ym` 范围，若与保单表覆盖月份不一致，UI 提示：

```
保单数据覆盖：2024-01 至 2026-04
人力数据覆盖：2026-01 至 2026-04
⚠️ 2024-01 至 2025-12 缺人力数据，活动率/产能模块此期间无法展示
```

## 5. SQLite Schema（草案）

```sql
CREATE TABLE fact_headcount (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  -- 时间
  period_ym  TEXT NOT NULL,        -- '2026-04'
  year       TEXT NOT NULL,
  month      INTEGER NOT NULL,
  -- 组织
  org        TEXT NOT NULL,
  sales_seq  TEXT,
  dept_code  TEXT,
  team_code  TEXT,
  -- 人员
  rank_grade TEXT,
  rank_level TEXT,
  staff_id   TEXT NOT NULL,
  staff_name TEXT,
  tenure_band TEXT,
  biz_mode   TEXT,
  hire_year  INTEGER,
  hire_month INTEGER,
  -- 在职
  hc_start   INTEGER NOT NULL DEFAULT 0,  -- 月初 0/1
  hc_end     INTEGER NOT NULL DEFAULT 0   -- 月末 0/1
);

CREATE UNIQUE INDEX ux_hc_period_staff ON fact_headcount(period_ym, staff_id);
CREATE INDEX ix_hc_period_org ON fact_headcount(period_ym, org);
CREATE INDEX ix_hc_staff ON fact_headcount(staff_id);
CREATE INDEX ix_hc_period ON fact_headcount(period_ym);
```

## 6. 派生指标定义

基于本表可计算的关键指标（详见各模块需求文档）：

### 6.1 月末在职人力

```sql
SELECT period_ym, org, SUM(hc_end) AS headcount_end
FROM fact_headcount
GROUP BY period_ym, org;
```

### 6.2 月均人力

```sql
SELECT period_ym, org,
       (SUM(hc_start) + SUM(hc_end)) / 2.0 AS headcount_avg
FROM fact_headcount
GROUP BY period_ym, org;
```

### 6.3 月度留存率

```sql
SELECT period_ym, org,
       1.0 * SUM(hc_end) / NULLIF(SUM(hc_start), 0) AS retention_rate
FROM fact_headcount
GROUP BY period_ym, org;
```

### 6.4 长险活动率（开放问题，分母待定）

```sql
-- 分子：本月出长险的不同人员数
WITH active_staff AS (
  SELECT DISTINCT year, month, staff_id
  FROM fact_premium
  WHERE term_type = '长期险' AND staff_id != ''
),
active_count AS (
  SELECT year, month, org, COUNT(DISTINCT staff_id) AS act_n
  FROM active_staff a
  JOIN fact_headcount h ON h.staff_id = a.staff_id
       AND h.year = a.year AND h.month = a.month
  GROUP BY year, month, org
),
-- 分母：月末在职（待用户确认是否改月均）
denom AS (
  SELECT year, month, org, SUM(hc_end) AS hc
  FROM fact_headcount
  GROUP BY year, month, org
)
SELECT a.year, a.month, a.org,
       1.0 * a.act_n / NULLIF(d.hc, 0) AS activity_rate
FROM active_count a
JOIN denom d USING(year, month, org);
```

### 6.5 队伍产能

```sql
-- 月度人均保费 = 折算保费总和 / 月末在职
SELECT p.year, p.month, p.org,
       SUM(p.zhsf_cents)/100.0 / NULLIF(SUM(h.hc_end), 0) AS per_capita_premium
FROM fact_premium p
JOIN fact_headcount h ON h.year = p.year
                     AND h.month = p.month
                     AND h.org = p.org
GROUP BY p.year, p.month, p.org;
```

> 详细口径与开放问题在模块 04、05 文档中讨论。

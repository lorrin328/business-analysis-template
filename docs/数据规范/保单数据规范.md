## 1. 文件命名

```
AI-经营分析业绩基表_<导出日期>_<导出时间序号>.xlsx
例：AI-经营分析业绩基表_20260426_00430282.xlsx
```

- 文件名仅作识别，导入逻辑不依赖
- 同名文件覆盖：以最后导入为准
- Excel 临时文件（`~$AI-...xlsx`）不入库，需 `.gitignore` 屏蔽

## 2. 字段清单

### 2.1 时间字段（必填）

| Excel 列名 | 别名 | 类型 | 入库列 | 示例 | 说明 |
|-----------|------|------|--------|------|------|
| `年` | - | TEXT | `year` | `2026` | 4 位字符串 |
| `月` | `年月` | INT | `month` | `4` 或 `2026-04` | 别名为 Date 类型时取 `getMonth()+1` |
| `季` | `年季` | INT | `quarter` | `2` 或 `2026-2` | 别名为字符串时 split `-` 取末段 |
| `日期` | `投保日期`/`签单日期`/`成交日期`/`年月日` | TEXT | `date` | `2026-04-15` | YYYY-MM-DD |
| `月标签` | - | TEXT | `month_label` | `2026年4月` | 缺失时自动衍生 |

### 2.2 时点字段（保留入库，部分模块使用）

| Excel 列名 | 入库列 | 类型 | 示例 | 业务含义 |
|-----------|--------|------|------|----------|
| `投保时间` | `app_date` | TEXT | `2026-04-15 10:23:00` | 客户投保时间 |
| `承保时间` | `underwrite_date` | TEXT | `2026-04-16 14:00:00` | 公司核保通过时间 |
| `入账时间` | `entry_date` | TEXT | `2026-04-17 09:00:00` | **当前主口径** |
| `回销时间` | `cancel_date` | TEXT | `2026-04-18 11:00:00` | 实际回款销账时间 |

> **统计口径**：保费时序按 `入账时间` 推导 year/month/quarter。激励方案口径预留：以 `承保时间` 为准，承保后 30 自然日内入账 + 再 15 自然日回销 + 无负金额。

### 2.3 核心维度（必填）

| Excel 列名 | 入库列 | 类型 | 示例 | 用途 |
|-----------|--------|------|------|------|
| `销售机构名称` | `org` | TEXT | `上海` | 与人力表关联键 |
| `业务模式` | `biz_mode` | TEXT | `OTO` | 与人力表关联键 |
| `长短险` | `term_type` | TEXT | `长期`/`短期`/`极短期` | 活动率分母；**极短期并入短期** |
| `是否商保年金产品` | `is_annuity` | TEXT | `Y`/`N` | 产品分类 |
| `缴费年限` | `pay_years` | TEXT | `10年`/`趸交` | 产品分类 |
| `保障年限` | `cov_years` | TEXT | `终身`/`20年` | 产品分类 |
| `产品设计分类` | `design_cat` | TEXT | `年金`/`重疾`/`寿险` | 产品结构图主维度 |

### 2.4 扩展维度（可选）

| Excel 列名 | 入库列 | 类型 | 用途 |
|-----------|--------|------|------|
| `人员工号` | `staff_id` | TEXT | **关联人力表** |
| `主管工号` | `supervisor_id` | TEXT | 主管层级聚合 |
| `经理工号` | `manager_id` | TEXT | 经理层级聚合 |
| `投保单号` | `policy_no` | TEXT | 唯一标识 |
| `自保件标记` | `self_mark` | TEXT | `Y`/`N` |
| `产品代码` | `product_code` | TEXT | 与 dim_product 关联 |
| `产品名称` | `product_name` | TEXT | 同上 |
| `是否个人养老金` | `is_pension` | TEXT | `Y`/`N` |

### 2.5 金额指标（必填）

| Excel 列名 | 入库列 | 类型 | 单位 | 示例 |
|-----------|--------|------|------|------|
| `期交保费` | `qj_cents` | INTEGER | 分 | Excel 元 × 100 |
| `年化规保` | `ghgb_cents` | INTEGER | 分 | 同上 |
| `折算保费` | `zhsf_cents` | INTEGER | 分 | 同上 |
| `价值规保` | `jzgb_cents` | INTEGER | 分 | 入库备用 |

> 金额一律以「分」存 INTEGER，规避浮点累计误差。查询时 `SUM(col) / 100.0` 还原元。

### 2.7 件数与附加险（v1.1 新增说明）

**件数定义**（用户 2026-04-26 决策）：

```sql
-- 月度件数（不含附加险）
COUNT(DISTINCT policy_no)
WHERE policy_no != ''
  AND zhsf_cents > 0
  AND <附加险过滤条件>   -- ⚠️ 待 Q10 确认
```

- 件数 = 不同保单号去重计数（**不是** `COUNT(*)` 行数）
- 同一保单可能含主险 + 附加险多行，**仅算一件**（COUNT DISTINCT 已天然去重）
- 但若需要单独排除附加险行（用于"主险件数"），需识别规则

**附加险识别（开放问题 Q10）**：

候选规则：
| 选项 | 实现 | 优点 | 缺点 |
|------|------|------|------|
| A | `is_pension = 0` | 简单 | 不准（is_pension 仅区分养老金） |
| B | `product_name NOT LIKE '%附加%'` | 直观 | 依赖产品名规范 |
| C | `product_code` 字典查表（`dim_product.is_rider`） | 准确 | 需维护字典 |

**默认（待用户提供规则前）**：选 B（关键字匹配），导入时打日志统计差异。

**长期目标**：建立 `dim_product` 维度表，包含 `product_code`, `product_name`, `is_rider`, `pay_type`, `term_type` 等字段，由用户提供产品字典或从历史数据归纳。

## 3. 校验规则

### 3.1 强制必需列

导入时逐一检查 §2.1（时间字段除月标签外）+ §2.3（核心维度）+ §2.5（金额指标）。任一缺失即报错并展示 Levenshtein 最相近的列名建议。

### 3.2 维度空值处理

- `null` / 空串 / `'nan'` / `'None'` → 入库填 `'未知'`
- 仅对 §2.3 核心维度（trimDim 函数）生效
- §2.4 扩展维度（trimEmpty 函数）→ 空串 → 不入库时填 `''`

### 3.3 金额异常处理

- 解析失败（NaN/Infinity）→ 入库 `0`
- 含逗号分隔（`1,234.56`）→ 先去逗号
- 负值合法（撤单/退保），不过滤

### 3.4 日期解析降级链

1. 单元格为 Date 对象 → 直接 `getFullYear/Month/Date`
2. 数字（Excel 序列号）→ `XLSX.SSF.parse_date_code()`
3. 字符串 `YYYY-MM-DD` / `YYYY/MM/DD` / `YYYY.MM.DD` → 正则匹配
4. 其他字符串 → `new Date()` 兜底
5. 全部失败 → `date = null, day = null`（仍允许入库，仅日明细不可用）

## 5. SQLite Schema（v0.4，对应当前实现）

```sql
CREATE TABLE fact_premium (
  id INTEGER PRIMARY KEY,
  -- 时间
  date TEXT, year TEXT NOT NULL, quarter INTEGER NOT NULL,
  month INTEGER NOT NULL, day INTEGER, month_label TEXT NOT NULL,
  -- 核心维度
  org TEXT, biz_mode TEXT,
  is_operating TEXT, is_dividend TEXT, innovate TEXT,  -- 停用
  term_type TEXT, is_annuity TEXT, pay_years TEXT,
  cov_years TEXT, design_cat TEXT,
  -- 扩展维度
  staff_id TEXT, supervisor_id TEXT, manager_id TEXT,
  policy_no TEXT, self_mark TEXT,
  app_date TEXT, underwrite_date TEXT,
  entry_date TEXT, cancel_date TEXT,
  product_code TEXT, product_name TEXT, is_pension TEXT,
  -- 金额（分）
  qj_cents INTEGER NOT NULL DEFAULT 0,
  ghgb_cents INTEGER NOT NULL DEFAULT 0,
  zhsf_cents INTEGER NOT NULL DEFAULT 0,
  jzgb_cents INTEGER NOT NULL DEFAULT 0
);

CREATE INDEX ix_ym   ON fact_premium(year, month);
CREATE INDEX ix_yq   ON fact_premium(year, quarter);
CREATE INDEX ix_ymd  ON fact_premium(year, month, day);
CREATE INDEX ix_org  ON fact_premium(org);
CREATE INDEX ix_mode ON fact_premium(biz_mode);
CREATE INDEX ix_dsg  ON fact_premium(design_cat);
CREATE INDEX ix_staff ON fact_premium(staff_id);  -- 新增，活动率/产能 JOIN 用
```

> P2 阶段会拆出 `dim_org` / `dim_product` 等维度表。届时 `org` 等字段可改为外键 ID，但保持向下兼容。

# 指标计算口径核对报告（v1.0.55）

本文档用于对当前经营分析看板涉及的指标口径、计算公式、数据来源和展示规则进行复核。文档依据当前代码实现整理，重点覆盖 KPI 概览、机构维度、平台趋势、产品结构、交期结构、队伍分析、目标设置、参数设置和 Excel 导出。

## 一、总原则

### 1. 单位口径

| 项目 | 口径 |
|---|---|
| 金额单位 | 页面、接口聚合表、Excel 导出统一为“万元” |
| 百分比展示 | 前端通常展示为百分数，机构维度达成率和同比保留 1 位小数 |
| 件数单位 | 产品结构、交期结构中的件数为“件” |
| 人力单位 | 队伍分析、人均保费分母为“人” |

### 2. 不可计算规则

| 指标 | 不可计算条件 | 展示规则 |
|---|---|---|
| 达成率 | 目标为空、缺失或为 0 | 展示 `--`、`-` 或配置缺失提示，不应默认解释为 0% |
| 同比 | 去年同期为空、缺失或为 0 | 展示 `--` 或 `-` |
| 活动率 | 在职人力为空、缺失或为 0 | 展示为空状态或 0，需结合模块说明判断 |
| 人均类指标 | 月均在职或月均活动人力为空、缺失或为 0 | 展示为空状态或 0 |

### 3. 时间口径

| 场景 | 当前实现 |
|---|---|
| KPI 概览期交保费 | 优先使用日级聚合；转型按转型日表最新日，经代按经代日表最新日；同比分别取上一年同月同日累计 |
| 平台趋势混合展示 | 当同时选择转型和经代时，按双方共同最早截止日截断，避免趋势曲线混用不同截止日 |
| 机构维度期交保费 | 按转型机构日级聚合表中各渠道最新截止日计算；经代不进入机构维度 |
| 价值、产品类指标 | 当前为月级聚合，按截至统计月累计 |
| 人力指标 | 当前为月级口径，按月初、月末、月均、活动人力计算 |

复核提示：如果后续要求“KPI 概览整体期交保费”也必须转型和经代使用完全相同截止日，需要把 `backend/db/repositories/kpi.py` 中 KPI 实绩取数逻辑从“各来源自身截止日”改为“共同截止日”。

## 二、基础公式

| 指标 | 公式 | 代码位置 |
|---|---|---|
| 达成率 | 实绩 / 目标 | `backend/metrics/formulas.py::achievement_rate`，前端也有局部展示计算 |
| 同比 | 本期累计 / 去年同期累计 - 1 | `backend/metrics/formulas.py::yoy_rate`，部分前端展示使用同等公式 |
| 环比 | 本期 / 上期 - 1 | `backend/metrics/formulas.py::mom_rate` |
| 序时进度 | 已过自然日 / 当年总自然日 | 机构维度前端用于颜色标识 |
| 进度偏差 | 达成率 - 序时进度 | `backend/metrics/formulas.py::progress_gap` |
| 活动率 | 活动人力 / 月均在职人力 | `backend/metrics/formulas.py::activity_rate` |
| 人均保费 | 月均期交保费 / 月均在职人力 | `backend/metrics/formulas.py::avg_premium`，前端 KPI 卡片实际展示为转型业务口径 |
| 人均产能 | 期交保费 / 月均活动人力 | `backend/metrics/formulas.py::avg_productivity` |
| 转化率 | 成交数 / 触达或线索数 | 公式已定义，当前不是主看板核心展示项 |
| 费用率 | 费用 / 保费 | 公式已定义，当前不是主看板核心展示项 |
| 投产比 | 产出 / 投入 | 公式已定义，当前不是主看板核心展示项 |

## 三、KPI 概览指标

### 1. 期交保费达成率

| 项目 | 口径 |
|---|---|
| 分子 | 经代期交保费 + OTO 期交保费 + 证保期交保费 + 蚁桥期交保费 |
| 分母 | 目标设置中 `qjPremium / 整体 / 年度目标` |
| 业务线拆分 | 经代、转型业务；转型业务 = OTO + 证保 + 蚁桥 |
| 数据源 | `agg_daily_performance`、`agg_jingdai_daily`；无完整日级数据时回退 `agg_performance`、`agg_jingdai` |
| 时间口径 | 当前 KPI 实绩按各来源自身日级截止日；同比按各来源上一年同月同日累计 |
| 前端展示 | 大字展示整体达成率；小字展示整体同比、经代达成率、经代同比、转型达成率、转型同比 |

公式：

```text
整体期交保费 = 经代期交保费 + OTO期交保费 + 证保期交保费 + 蚁桥期交保费
整体达成率 = 整体期交保费 / 期交保费整体目标
经代达成率 = 经代期交保费 / 期交保费经代目标
转型达成率 = 转型期交保费 / 期交保费转型目标
整体同比 = 本年整体期交保费累计 / 去年整体同截止日累计 - 1
经代同比 = 本年经代期交保费累计 / 去年经代同截止日累计 - 1
转型同比 = 本年转型期交保费累计 / 去年转型同截止日累计 - 1
```

特殊规则：

1. 转型同比为业务线维度比较，使用 OTO、证保、蚁桥合计，因此包含 2025 年蚁桥未归属机构业绩。
2. 机构维度同比为机构维度比较，2025 年蚁桥未归属机构业绩不纳入机构维度统计。
3. 若目标为 0 或未配置，达成率不应作为有效经营结果解读。

### 2. 价值达成率

| 项目 | 口径 |
|---|---|
| 分子 | 经代 + OTO + 证保 + 蚁桥价值保费；经代价值数据未接入前，经代实绩显式按 0 展示 |
| 分母 | 目标设置中 `value / 整体 / 年度目标` |
| 数据源 | `agg_value_data` |
| 时间口径 | 月级累计，截至当前统计月 |
| 经代口径 | 经代纳入价值达成率口径；当前没有经代价值数据表，因此实绩暂为 0，待数据表接入后自动纳入 |

公式：

```text
价值保费 = 经代价值保费 + OTO价值保费 + 证保价值保费 + 蚁桥价值保费
价值达成率 = 价值保费 / 价值保费整体目标
```

### 3. 长险活动率

| 项目 | 口径 |
|---|---|
| 分子 | 当前统计月长险活动人力 |
| 分母 | 当前统计月月均在职人力 |
| 数据源 | `agg_hr_data` |
| 时间口径 | 月级，取当前统计月 |
| 同比展示 | 本年活动率 - 去年同期活动率，单位为百分点 pp |

公式：

```text
月均在职人力 = (月初人力 + 月末人力) / 2
长险活动率 = 活动人力 / 月均在职人力
长险活动率同比 = 本年同期活动率 - 去年同期活动率
```

注意：长险活动率同比不是相对增速，不使用“本年活动率 / 去年活动率 - 1”。

### 4. 商保年金达成率

| 项目 | 口径 |
|---|---|
| 分子 | 被参数设置标识为“商保年金”的产品期交保费，含转型和经代 |
| 分母 | 目标设置中 `shangbao / 整体 / 年度目标` |
| 数据源 | 转型来自 `agg_org_performance.product_annuity`，经代来自 `agg_jingdai.product_annuity` |
| 参数来源 | `product_config` 中 `is_annuity = Y` |
| 时间口径 | 月级累计，截至当前统计月 |

公式：

```text
商保年金实绩 = 转型商保年金期交保费 + 经代商保年金期交保费
商保年金达成率 = 商保年金实绩 / 商保年金整体目标
```

### 5. 保障类产品达成率

| 项目 | 口径 |
|---|---|
| 分子 | 被参数设置标识为“保障类”的产品期交保费，含转型和经代 |
| 分母 | 目标设置中 `baozhang / 整体 / 年度目标` |
| 数据源 | 转型来自 `agg_org_performance.product_protection`，经代来自 `agg_jingdai.product_protection` |
| 参数来源 | `product_config` 中 `is_protection = Y` |
| 时间口径 | 月级累计，截至当前统计月 |

公式：

```text
保障类产品实绩 = 转型保障类产品期交保费 + 经代保障类产品期交保费
保障类产品达成率 = 保障类产品实绩 / 保障类产品整体目标
```

注意：如果没有配置产品标识或未重新计算聚合，实绩可能为 0；这代表参数/聚合状态，不应直接解释为业务没有保障类产品。

### 6. 10年期产品达成率

| 项目 | 口径 |
|---|---|
| 分子 | 转型 10 年及以上期交产品 + 经代 10 年及以上交期产品 |
| 分母 | 目标设置中 `tenYear / 整体 / 年度目标` |
| 转型来源 | `agg_org_performance.product_10year` |
| 经代来源 | `agg_payment_period` 中 `business_type = 经代` 且 `category = 10年及以上` |
| 时间口径 | 月级累计，截至当前统计月 |

公式：

```text
10年期产品实绩 = 转型10年期产品期交保费 + 经代10年及以上期交保费
10年期产品达成率 = 10年期产品实绩 / 10年期产品整体目标
```

### 7. 长险期交达成率

| 项目 | 口径 |
|---|---|
| 分子 | 长险期交保费，含转型和经代 |
| 分母 | 沿用期交保费目标，即 `qjPremium / 整体 / 年度目标` |
| 数据源 | `agg_longterm_qj` |
| 时间口径 | 当前代码按业务来源日级截止日过滤；无日级数据时按月级累计 |

公式：

```text
长险期交实绩 = 转型长险期交保费 + 经代长险期交保费
长险期交达成率 = 长险期交实绩 / 期交保费整体目标
长险期交同比 = 本年长险期交累计 / 去年同期长险期交累计 - 1
```

长险判定规则：

| 业务类型 | 判定规则 |
|---|---|
| 转型 | `长短险` 属于长期类，或产品代码标准化后为 `4281`；长短险为空时，缴费年限大于等于 2 视为长期 |
| 经代 | 排除“当前缴别大类=期交 且 缴费年限=1”的短期口径，其余纳入长险期交 |

### 8. 人均保费

| 项目 | 口径 |
|---|---|
| 分子 | 转型业务期交保费累计，不含经代 |
| 分母 | 转型业务月均在职人力 |
| 数据源 | 分子来自 KPI 期交保费，分母来自 `agg_hr_data` |
| 时间口径 | 保费按日级或月级累计；人力为月级 |

公式：

```text
统计月数 = 当前人力数据有效月份数
月均期交保费 = 转型期交保费累计 / 统计月数
月均在职人力 = 各业务线月均在职人力合计
人均保费 = 月均期交保费 / 月均在职人力
```

注意：该指标当前为转型业务队伍经营指标，不含经代。

## 四、机构维度指标

机构维度仅统计 OTO、证保、蚁桥等转型业务中有明确机构归属的数据，经代不进入机构维度。2025 年蚁桥未归属机构业绩不纳入机构维度统计和机构维度同比。

### 1. 展示层级

| 状态 | 展示规则 |
|---|---|
| 默认收起 | 按机构汇总展示 |
| 点击展开 | 展示机构下 OTO、证保、蚁桥明细，并在同机构下展示小计 |

### 2. 机构维度指标表

| 指标 | 分子 | 分母 | 同比 |
|---|---|---|---|
| 期交保费 | 机构内所选期间期交保费 | 机构期交保费目标 | 本年同机构同业务模式同期间 / 去年同期 - 1 |
| 价值保费 | 机构内所选期间价值保费 | 机构价值保费目标 | 本年同机构同业务模式同期间 / 去年同期 - 1 |
| 长险期交（年度） | 机构内年度长险期交保费 | 机构期交保费年度目标 | 当前不展示同比 |
| 10年期产品 | 机构内所选期间 10 年期产品期交保费 | 机构 10 年期产品目标 | 当前不展示同比 |
| 商保年金 | 机构内所选期间商保年金期交保费 | 机构商保年金目标 | 当前不展示同比 |
| 保障类产品 | 机构内所选期间保障类产品期交保费 | 机构保障类目标 | 当前不展示同比 |

机构汇总公式：

```text
机构汇总实绩 = 该机构下 OTO + 证保 + 蚁桥实绩
机构汇总目标 = 该机构下已配置目标的业务模式目标合计
机构汇总达成率 = 机构汇总实绩 / 机构汇总目标
机构汇总同比 = 机构汇总本年实绩 / 机构汇总去年同期实绩 - 1
```

总计行公式：

```text
总计实绩 = 所有展示机构、展示业务模式实绩合计
总计目标 = 所有展示机构、展示业务模式目标合计
总计达成率 = 总计实绩 / 总计目标
总计同比 = 总计本年实绩 / 总计去年同期实绩 - 1
```

### 3. 颜色规则

| 指标 | 规则 |
|---|---|
| 达成率低于序时进度 | 绿色 |
| 达成率高于或等于序时进度且低于 100% | 淡红色 |
| 达成率大于或等于 100% | 红色 |
| 同比为负 | 绿色 |
| 同比大于或等于 0 且低于 10% | 淡红色 |
| 同比大于或等于 10% | 红色 |

序时进度公式：

```text
序时进度 = 当年已过自然日 / 当年总自然日
```

## 五、平台趋势指标

| 指标 | 字段 | 口径 |
|---|---|---|
| 期交保费 | `qj_premium` | 期交保费累计 |
| 规模保费 | `gm_premium` | 规模或年化规保累计 |
| 折算保费 | `zs_premium` | 折算保费累计 |

数据源：

| 业务类型 | 月级来源 | 日级来源 |
|---|---|---|
| 转型 | `agg_performance` | `agg_daily_performance` |
| 经代 | `agg_jingdai` | `agg_jingdai_daily` |

公式：

```text
期间累计值 = 按所选业务线、所选保费口径，对月度或日度数据逐期累计
```

特殊规则：

1. 年度视图展示月度累计。
2. 季度视图展示季度内按日累计。
3. 月度视图展示月内按日累计。
4. 同时选择转型和经代时，按共同最早日级截止日截断，避免不同来源截止日不一致。
5. 如果缺少真实日级数据，接口返回空状态，不伪造日累计曲线。

## 六、产品结构指标

### 1. 产品设计分类

| 项目 | 口径 |
|---|---|
| 数据源 | `agg_product_structure` |
| 金额 | 所选分类下保费合计 |
| 件数 | 所选分类下件数合计 |
| 展示 | 默认取保费排序前 12 项 |

### 2. 产品明细结构

| 项目 | 口径 |
|---|---|
| 转型来源 | 原始表 `performance` |
| 经代来源 | 原始表 `jingdai` |
| 转型产品字段 | `产品类型` |
| 经代产品字段 | `产品名称` |
| 金额字段 | 期交保费或规模保费 |
| 件数字段 | 转型优先 `承保件数`，缺失时默认按 1；经代按记录数计件 |

公式：

```text
产品保费 = 同一产品分类或产品名称下保费求和 / 10000
产品件数 = 同一产品分类或产品名称下件数求和
产品占比 = 单项产品保费 / 当前筛选范围产品保费合计
```

### 3. 各业务模式前三名产品

| 项目 | 口径 |
|---|---|
| 数据源 | 原始表 `performance`、`jingdai` |
| 业务模式 | OTO、证保、蚁桥、经代 |
| 转型归类 | 原始 `业务模式` 中“证券”归入证保，“网服”归入蚁桥 |
| 产品字段 | 转型优先 `产品名称`，缺失时依次使用 `产品类型`、`产品代码`；经代使用 `产品名称` |
| 金额字段 | 固定使用期交保费，不受产品结构饼图保费类型切换影响 |

公式：

```text
业务模式期交保费 = 同一业务模式下期交保费合计 / 10000
单产品期交保费 = 同一业务模式、同一产品下期交保费合计 / 10000
前三名产品 = 按单产品期交保费降序取前三项；不足三项时按实际项数展示
模式内占比 = 单产品期交保费 / 业务模式期交保费
```

## 七、交期结构指标

| 项目 | 口径 |
|---|---|
| 数据源 | `agg_payment_period` |
| 金额口径 | `qj_premium` 或 `gm_premium` |
| 件数口径 | `count` |
| 维度 | 趸交、短期险、3 年交、5 年交、10 年及以上等交期分类 |

公式：

```text
交期保费 = 所选年份、月份、业务类型、机构、业务线下同一交期分类保费合计
交期件数 = 所选范围下同一交期分类件数合计
交期保费占比 = 单一交期分类保费 / 所选范围交期保费合计
交期件数占比 = 单一交期分类件数 / 所选范围交期件数合计
```

注意：经代源表如无件数字段，经代件数可能为 0 或按聚合逻辑缺省展示，复核时应以保费结构为主。

## 八、队伍分析指标

| 指标 | 公式 | 数据源 | 业务范围 |
|---|---|---|---|
| 月初人力 | 源表月初在职人力 | `agg_hr_data` | 转型业务 |
| 月末人力 | 源表月末在职人力 | `agg_hr_data` | 转型业务 |
| 月均在职人力 | (月初人力 + 月末人力) / 2 | `agg_hr_data` | 转型业务 |
| 活动人力 | 源表活动人力 | `agg_hr_data` | 转型业务 |
| 活动率 | 活动人力 / 月均在职人力 | `agg_hr_data` | 转型业务 |
| 人均保费 | 月均期交保费 / 月均在职人力 | 期交保费 + `agg_hr_data` | 转型业务 |
| 人均产能 | 期交保费 / 月均活动人力 | 期交保费 + `agg_hr_data` | 转型业务 |

注意：经代当前无队伍人力基表，不进入队伍分析。

### 1. 队伍结构与产能分析（试运行）

| 展示项 | 当前口径 | 数据源 |
|---|---|---|
| 样本人数 | `hr_data` 中当月月初或月末在职人员数量，剔除“小计”和空人员代码 | `hr_data` |
| 司龄段结构 | 按 `月末司龄区间` 汇总月末在职人数、占比、正产能人数、期交保费、人均保费 | `hr_data` LEFT JOIN `performance` |
| 职等结构 | 按 `职等` 汇总月末在职人数、占比、正产能人数、期交保费、人均保费 | `hr_data` LEFT JOIN `performance` |
| 产能段结构 | 按人员月产能分为 0及以下、0-0.5万、0.5-1万、1-3万、3-5万、5-10万、10万以上 | `hr_data` LEFT JOIN `performance` |
| P25/P50/P75 | 按人员产能升序后线性插值计算，默认纳入零/负产能人员 | `hr_data` LEFT JOIN `performance` |
| ≥P25/≥P50/≥P75人数 | 人员产能达到对应分位阈值及以上的人数 | `hr_data` LEFT JOIN `performance` |
| 分机构分位数 | 按机构分别计算样本、零/负产能占比、P25/P50/P75 及对应达标人数 | `hr_data` LEFT JOIN `performance` |
| 近 6 个月分位趋势 | 按同一口径逐月计算样本人数、零/负产能占比、P25/P50/P75 及对应达标人数 | `hr_data` LEFT JOIN `performance` |

计算细则：

1. 人员匹配：`hr_data.人员代码 = performance.人员工号`，并统一处理 `1001` 与 `1001.0` 等数字型工号格式。
2. 时间匹配：月度同一年、同一月匹配；季度/年度按所选期间内月份累计。
3. 产能口径：月度产能 = 当月个人期交保费 / 10000；季度/年度产能 = 期间内个人累计期交保费 / 10000，单位为万元。
4. 零/负产能人员：人员月产能小于等于 0 的人员，默认纳入样本和分位数。
5. 正产能人员：人员月产能大于 0 的人员。
6. 业务模式映射：证券=证保，网服=蚁桥，OTO 保持不变。
7. 人数去重：季度/年度同一期间内同一人员只计 1 人；机构、司龄、职等取期间最后一个在职月份。
8. 业务模式筛选：“全部”按 OTO、证保、蚁桥合计；选择单一模式时仅计算对应业务模式人员样本。

说明：该模块已不再使用汇总表反推，正式读取 SQLite 原始表进行交叉计算。经代当前无队伍人力基表，不进入队伍结构与产能分析。

## 十二、建议复核重点

以下事项建议你重点确认，确认后可作为后续新增模块的固定底座口径：

1. KPI 概览期交保费是否继续允许“经代、转型各自截止日”，还是统一改为“共同最早截止日”。
2. 长险期交达成率分母已按你的意见继续使用期交保费目标，是否长期固定。
3. 经代价值保费已纳入价值达成率口径；当前经代价值数据表尚未接入，实绩暂按 0 展示。
4. 保障类产品和商保年金完全依赖参数设置，是否需要增加“未配置产品数/已配置产品数”的提示。
5. 机构维度不纳入未归属机构业绩，但 KPI 转型业务线同比纳入未归属蚁桥业绩，该差异是否作为正式口径固定。

# v1.0.49 发布说明

## 版本信息

- 版本号：v1.0.49
- 发布日期：2026-05-25
- 类型：fix / data-cleaning

## 问题背景

参数设置中可能出现 `nan` 无效产品行。根因是 Excel 空单元格被 pandas 读取为缺失值后，在部分清洗路径中被转成字符串 `"nan"`，进而写入 `product_config`。

## 修复内容

1. 产品代码为 `nan`、`none`、`null` 时统一视为空值，不再写入参数配置。
2. 产品名称为 `nan`、`none`、`null` 时统一清空，避免页面展示无效名称。
3. 参数设置读取时自动清理历史 `nan` 产品代码记录。
4. 保存产品参数时同步执行产品代码与名称清洗。

## 影响范围

- 影响模块：参数设置、产品参数自动抽取、产品参数保存。
- 不改变已有正常产品代码、产品名称和商保年金/保障类标识。
- 不涉及数据库结构迁移。

## 验证结果

- 新增测试覆盖：产品代码/产品名称 `nan` 清洗。
- 本地回归测试：`168 passed`

## 部署提示

部署后打开“参数设置”页面即可触发历史无效 `nan` 产品记录清理。

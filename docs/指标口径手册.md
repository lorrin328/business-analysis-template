## 2. 时间口径

### 2.1 保费统计时间

**按 `入账时间` 统计**。`年`/`年月`/`年季`/`年月日` 均按 `入账时间` 推导。

### 2.2 基准年

`BASE_YEAR = String(new Date().getFullYear())`，系统当前年份自动滚动，**不硬编码、不允许 UI 选择**。

| 当前日期 | 基准年 | 含义 |
|---------|--------|------|
| 2026-04-27 | 2026 | 同比以 2026 年为基准 |
| 2027-01-01 起 | 2027 | 自动切换 |

### 2.3 激励方案口径（预留）

以 **`承保时间`** 为准，承保后 **30 个自然日（含）** 内入账，再过 **15 个自然日（含）** 回销，且 **未发生负金额**（无撤单/退保）。

> 注：该口径尚未在看板中实现，仅作为预留规则入库。

## 3. 保费类指标

### 3.1 三口径定义

| 指标 | 入库列 | 单位 | 说明 |
|------|--------|------|------|
| **期交保费** | `qj_cents` | 元 | 非趸交长险 + 短险 + 长险趸交 × 0.1 |
| **折算保费** | `zhsf_cents` | 元 | 长险趸交×0.1 + [3,5)年×0.3 + [5,10)年×0.5 + 10年及以上×1 + 短险×0.1 |
| **年化规保** | `ghgb_cents` | 元 | 年化规模保费 |

> 金额统一按「分」存 INTEGER：`元 × 100` 后 `Math.round()`。查询时 `SUM(col) / 100.0` 还原。

### 3.2 累计保费（主图）

```sql
SELECT year, month, SUM(zhsf_cents) / 100.0 AS amount
FROM fact_premium
WHERE <筛选条件>
GROUP BY year, month
ORDER BY year, month;
```

JS 端按 year 分组后逐月累加得到 cumulative 数组。

### 3.3 同比（YoY）

```
yoy(year, period) = (current_cum - base_cum) / base_cum × 100%
base_year = new Date().getFullYear()
```

## 4. 人力类指标

### 4.1 月末在职人力

```sql
SELECT period_ym, org, SUM(hc_end) AS headcount
FROM fact_headcount
GROUP BY period_ym, org;
```

### 4.2 月初在职人力

```sql
SELECT period_ym, org, SUM(hc_start) AS headcount
FROM fact_headcount
GROUP BY period_ym, org;
```

### 4.3 月均在职人力（默认口径）

```sql
SELECT period_ym, org,
       (SUM(hc_start) + SUM(hc_end)) / 2.0 AS hc_avg
FROM fact_headcount
GROUP BY period_ym, org;
```

> 模块 04（活动率）、模块 05（产能 P1/P3）默认采用 **月均**。

### 4.4 外勤管理职判定

以 `rank_level`（职级）**白名单**判定，**排除「客户经理」系列营销岗**：

```js
const MGMT_RANKS = new Set([
  // 主管序列
  '储备主管','见习主管','初级主管','中级主管','高级主管',
  '资深主管一级','资深主管二级',
  // 经理序列（营销岗除外）
  '储备经理','见习经理','初级经理','高级经理','资深经理',
  // 总监
  '总监'
]);
```

**归入「一线营销员」的职级示例**：客户专员一级、客户经理一级、资深客户经理、网服专员、创新专员等（所有不在白名单中的职级）。

## 5. 活动率指标

### 5.1 活动人力

当月有保单（`zhsf_cents > 0`）的不重复员工数。

```sql
SELECT p.year, p.month, h.org,
       COUNT(DISTINCT p.staff_id) AS act_n
FROM fact_premium p
JOIN fact_headcount h
  ON h.staff_id = p.staff_id
 AND h.year = p.year
 AND h.month = p.month
WHERE p.staff_id != ''
  AND p.zhsf_cents > 0
GROUP BY p.year, p.month, h.org;
```

### 5.2 长险活动人力

当月有**长险**保单（`term_type = '长期'`）的员工，是活动人力的子集。

```sql
SELECT p.year, p.month, h.org,
       COUNT(DISTINCT p.staff_id) AS chx_act_n
FROM fact_premium p
JOIN fact_headcount h
  ON h.staff_id = p.staff_id
 AND h.year = p.year
 AND h.month = p.month
WHERE p.term_type = '长期'
  AND p.staff_id != ''
  AND p.zhsf_cents > 0
GROUP BY p.year, p.month, h.org;
```

> `term_type` 实际值：「长期」「短期」「极短期」。极短期并入短期。

### 5.3 长险活动率（默认口径）

```sql
WITH chx_active AS (
  SELECT p.year, p.month, h.org,
         COUNT(DISTINCT p.staff_id) AS chx_act_n
  FROM fact_premium p
  JOIN fact_headcount h
    ON h.staff_id = p.staff_id
   AND h.year = p.year
   AND h.month = p.month
  WHERE p.term_type = '长期'
    AND p.staff_id != ''
    AND p.zhsf_cents > 0
  GROUP BY p.year, p.month, h.org
),
hc_avg AS (
  SELECT year, month, org,
         (SUM(hc_start) + SUM(hc_end)) / 2.0 AS hc_avg
  FROM fact_headcount
  GROUP BY year, month, org
)
SELECT t.year, t.month, t.org,
       1.0 * COALESCE(a.chx_act_n, 0) / NULLIF(t.hc_avg, 0) AS chx_activity_rate
FROM hc_avg t
LEFT JOIN chx_active a USING(year, month, org);
```

**关键规则**：
- 分母固定为 **月均在职人力**，不提供口径切换
- 用 `LEFT JOIN`，避免某机构当月零活动时丢行
- 分母 `NULLIF` 保护，避免除 0
- 同比用**百分点差**（pp），不用相对增速

## 6. 产能指标

### 6.1 三定义汇总

| 编号 | 指标 | 分子 | 分母 | 适用场景 |
|------|------|------|------|---------|
| **P1** | 人均保费 | 期交保费 `qj_cents` | 月均在职人力 | 全员人均产出 |
| **P2** | 人均产能 | 期交保费 `qj_cents` | **月均活动人力** | 仅出单人员人均 |
| **P3** | 人均件数 | `COUNT DISTINCT policy_no` | 月均在职人力 | 件均频次 |

### 6.2 月均活动人力（P2 分母）

- **单月**：当月实际有保单的不重复员工数（`COUNT DISTINCT staff_id WHERE zhsf_cents > 0`）
- **多月**：各月活动人力之和 / 月数

> 例：Q1 人均产能 = Q1 期交保费累计 / ((1月活动人力 + 2月活动人力 + 3月活动人力) / 3)

### 6.3 P1 人均保费 SQL

```sql
WITH premium_by_month AS (
  SELECT year, month, org, SUM(qj_cents) / 100.0 AS qj_total
  FROM fact_premium WHERE <筛选> GROUP BY year, month, org
),
hc_avg_by_month AS (
  SELECT year, month, org, (SUM(hc_start) + SUM(hc_end)) / 2.0 AS hc_avg
  FROM fact_headcount GROUP BY year, month, org
)
SELECT p.year, p.month, p.org,
       p.qj_total / NULLIF(h.hc_avg, 0) AS per_capita_premium
FROM premium_by_month p
JOIN hc_avg_by_month h USING(year, month, org);
```

### 6.4 P2 人均产能 SQL

```sql
WITH premium_by_month AS (
  SELECT year, month, org, SUM(qj_cents) / 100.0 AS qj_total
  FROM fact_premium WHERE staff_id != '' AND zhsf_cents > 0
  GROUP BY year, month, org
),
active_hc AS (
  SELECT year, month, org, COUNT(DISTINCT staff_id) AS act_n
  FROM fact_premium WHERE staff_id != '' AND zhsf_cents > 0
  GROUP BY year, month, org
)
SELECT p.year, p.month, p.org,
       p.qj_total / NULLIF(a.act_n, 0) AS per_active_premium
FROM premium_by_month p
JOIN active_hc a USING(year, month, org);
```

### 6.5 P3 人均件数 SQL

> ✅ v1 不区分附加险，全部 `policy_no` 计入。

```sql
WITH cnt_by_month AS (
  SELECT year, month, org, COUNT(DISTINCT policy_no) AS policy_cnt
  FROM fact_premium WHERE policy_no != '' AND zhsf_cents > 0
  GROUP BY year, month, org
),
hc_avg_by_month AS (
  SELECT year, month, org, (SUM(hc_start) + SUM(hc_end)) / 2.0 AS hc_avg
  FROM fact_headcount GROUP BY year, month, org
)
SELECT c.year, c.month, c.org,
       1.0 * c.policy_cnt / NULLIF(h.hc_avg, 0) AS per_capita_count
FROM cnt_by_month c
JOIN hc_avg_by_month h USING(year, month, org);
```

## 8. 数据处理规则

### 8.1 小计行过滤（必做）

人力表导入时，任何维度等于 **'小计'** 即跳过：

```sql
WHERE org = '小计' OR sales_seq = '小计' OR rank_grade = '小计'
   OR rank_level = '小计' OR biz_mode = '小计' OR tenure_band = '小计'
```

不过滤将导致人力被**重复计 2 倍**。

### 8.2 增量导入策略

同 `(period_ym, staff_id)` 重复时，采用**整月覆盖**：

```sql
BEGIN;
DELETE FROM fact_headcount WHERE period_ym IN (...);
INSERT INTO fact_headcount (...) VALUES (...), (...), ...;
COMMIT;
```

### 8.3 工号变更

**视为不同人**，不做身份合并。旧工号 = 流失，新工号 = 新增。

### 8.4 列名别名系统

| 标准列名 | 别名 | 说明 |
|----------|------|------|
| `月` | `年月` | Date 类型，取月份 |
| `季` | `年季` | 字符串如 `'2024-1'` |
| `日期` | `年月日` | Date 类型 |

## 10. 待确认口径

以下 3 项口径尚未确认，数据分析显示可能对指标产生实质影响：

### 10.1 自保件是否纳入统计

| 数据 | 数值 |
|------|------|
| 自保件（self_mark=Y） | 1,080 条（1.57%） |
| 自保件期交保费 | **486 万元** |
| 自保件折算保费 | **246 万元** |

当前所有模块 **未过滤自保件**。如需排除，平台趋势、产品结构、KPI 均会下降约 1.5%。

### 10.2 工号为空保单的处理

| 数据 | 数值 |
|------|------|
| 工号为空 | 13,451 条（**19.54%**） |
| 空工号期交保费 | **2,859 万元** |

当前处理不一致：
- **平台趋势**（模块 01）：无 staff_id 过滤，空工号保单**全额计入**
- **活动率/产能**（模块 04/05）：`staff_id != ''`，空工号保单**完全排除**

**影响**：模块间数据无法互相对上（平台趋势的保费 ≠ 活动率能解释的人力产出）。

### 10.3 负金额（撤单/退保）的处理

| 数据 | 数值 |
|------|------|
| 负金额保单 | 12,624 条 |
| 负金额合计 | **-1.77 亿元** |

当前处理不一致：
- **活动率模块**：`zhsf_cents > 0` 过滤负金额
- **平台趋势模块**：未明确是否过滤
- **产品结构模块**：饼图保留负值（显示为退保/撤单）

**影响**：同一批撤单在不同模块中呈现不同结果，累计保费的真实性受影响。

# v1.0.89 发布说明

## 版本信息

- 版本号：v1.0.89
- 发布日期：2026-05-29
- 类型：荣誉体系追踪分析看板重构

## 本次变更

1. 新增 `/api/honor/dashboard` 聚合接口，由后端统一组织荣誉体系看板数据。
2. 星钻联盟页面重构为经营追踪型看板，不再以原始表格平铺为主。
3. 新增和强化五类分析视角：
   - 机构追踪：按机构和项目观察会员率、获钻、扣减和测算奖励。
   - 项目分析：聚焦 OTO、证保两个星钻适用项目。
   - 专员级：单独观察非管理职人群的会员转化和月度扣减。
   - 管理职：单独观察主管、经理层级的荣誉沉淀和带动情况。
   - 月度预警：集中展示扣减、保号、未达标和数据异常。
4. 月度预警补充人员姓名、项目、层级、当前钻石、标保、长险件数和建议动作，便于逐人跟进。
5. 数据适配页调整为“数据审计”，定位为字段支撑性说明，避免与经营结果混淆。

## 口径边界

- 星钻荣誉体系仍仅覆盖 OTO、证保；经代和蚁桥（网服）不纳入星钻计算。
- 当前重构重点是追踪、分析和展示体系，不改变既有个人星钻 MVP 计算规则。
- 团队星钻复杂规则仍按后续规则治理，不在本版本强行扩展。

## 验证结果

- 补充 `/api/honor/dashboard` 接口测试，覆盖机构、项目、专员级、管理职和月度预警输出。
- 补充前端静态校验，确认页面包含总览驾驶舱、机构追踪、项目分析、专员级、管理职、月度预警和数据审计。
- 全量自动化测试通过后发布。

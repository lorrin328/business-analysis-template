## 1. 背景

"活动率"是保险营销团队的核心人效指标，反映团队中"在做事"的人占比。本模块聚焦**长险活动率**（长险活动人力占在职人力之比），用于跨机构、跨时间对比。

**术语规范**（用户 2026-04-26 决策）：
- 「活动人力」（取代旧称「活跃人力」）：当月有保单（zhsf > 0）的员工
- 「长险活动人力」：当月有长险保单的员工，是活动人力的子集
- 「在职人力」：当月有人力档案的员工（hc_end=1 或 hc_start=1）

## 2. 范围

### 2.1 包含

- 长险活动率月度趋势（按机构对比）
- 同比对比（与基准年同期，基准年 = 系统当前年份，自动滚动）
- 多维度切分：按机构、销售序列、职等、司龄、外勤管理职
- 活动率分布直方图（看团队整体水平分布）
- KPI：当月活动率、环比、同比

### 2.2 不包含

- 短险活动率（用户聚焦长险，可后续扩展）
- 个人级活动详情
- 活动人次（按件数 vs 按人）—— 仅按人
- 滚动 12 月活动率（MAT，v1 不做）

## 3. 数据需求

### 3.1 分子：长险活动人力

```sql
SELECT year, month, org, COUNT(DISTINCT staff_id) AS chx_act_n
FROM fact_premium
WHERE term_type = '长期险'   -- ⚠️ 实际值待数据校验确认（见 Q11）
  AND staff_id != ''
  AND zhsf_cents > 0  -- 排除负值（撤单）
GROUP BY year, month, org;
```

> `chx_act_n` = 长险活动人力数

### 3.2 分母：月均在职人力（已确认）

**用户决策（2026-04-26）**：分母统一采用「月均在职人力」，不再提供口径切换。

```sql
SELECT year, month, org,
       (SUM(hc_start) + SUM(hc_end)) / 2.0 AS hc_avg
FROM fact_headcount
GROUP BY year, month, org;
```

### 3.3 维度切分

按 `fact_headcount` 的维度分组：
- `org`（销售机构）
- `sales_seq`（销售序列）
- `rank_grade`（职等）
- `tenure_band`（司龄区间）
- `biz_mode`（业务模式）
- **`mgmt_level`（外勤管理职聚合，新增）**：'外勤管理职' / '一线营销员'

## 4. 计算口径

### 4.1 核心 SQL（月均口径，已确认）

```sql
WITH chx_active AS (
  -- 长险活动人力（按工号当月归属机构）
  SELECT p.year, p.month, h.org, h.sales_seq, h.mgmt_level,
         COUNT(DISTINCT p.staff_id) AS chx_act_n
  FROM fact_premium p
  JOIN fact_headcount h
    ON h.staff_id = p.staff_id
   AND h.year = p.year
   AND h.month = p.month
  WHERE p.term_type = '长期险'
    AND p.staff_id != ''
    AND p.zhsf_cents > 0
  GROUP BY p.year, p.month, h.org, h.sales_seq, h.mgmt_level
),
hc_total AS (
  -- 月均在职人力
  SELECT year, month, org, sales_seq, mgmt_level,
         (SUM(hc_start) + SUM(hc_end)) / 2.0 AS hc_avg
  FROM fact_headcount
  GROUP BY year, month, org, sales_seq, mgmt_level
)
SELECT t.year, t.month, t.org, t.sales_seq, t.mgmt_level,
       COALESCE(a.chx_act_n, 0) AS chx_active_n,
       t.hc_avg AS total_avg,
       1.0 * COALESCE(a.chx_act_n, 0) / NULLIF(t.hc_avg, 0) AS chx_activity_rate
FROM hc_total t
LEFT JOIN chx_active a USING(year, month, org, sales_seq, mgmt_level)
ORDER BY t.year, t.month, t.org;
```

**注意**：
- 用 `LEFT JOIN`，避免某机构当月零活动时丢行
- 分母 `NULLIF` 保护，避免除 0
- 出单时用工号关联人力表（同月），跨月不算
- 分子要求 `zhsf_cents > 0` 过滤撤单
- **分母为月均**，不再支持月末/月初切换

### 4.2 多月趋势

按 `period_ym` 排序后展开为折线图。

### 4.3 同比

```
yoy = (curr_rate - base_rate) × 100   (绝对差，单位 pp)
base_year = new Date().getFullYear()  -- 基准年自动滚动
```

> 活动率本身是百分比，同比用"百分点差"更直观，不用相对增速。

### 4.4 活动率分布

按 `org` 分组后，用 `(chx_act_n, hc_avg)` 计算每个机构的活动率，绘制直方图（横轴活动率、纵轴机构数）。

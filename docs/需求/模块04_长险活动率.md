## 1. 背景

"活动率"是保险营销团队的核心人效指标，反映团队中"在做事"的人占比。本模块聚焦**长险活动率**（长期险出单人占在职人力之比），用于跨机构、跨时间对比。

## 2. 范围

### 2.1 包含

- 长险活动率月度趋势（按机构对比）
- 同比对比（与基准年同期）
- 多维度切分：按机构、销售序列、职等、司龄
- 活动率分布直方图（看团队整体水平分布）
- KPI：当月活动率、环比、同比

### 2.2 不包含

- 短险活动率（用户聚焦长险，可后续扩展）
- 个人级活动详情
- 活动人次（按件数 vs 按人）—— 仅按人

## 3. 数据需求

### 3.1 分子：长险出单人数

```sql
SELECT year, month, org, COUNT(DISTINCT staff_id) AS active_staff_n
FROM fact_premium
WHERE term_type = '长期险'   -- ⚠️ 需确认实际值（可能是"长期"/"长险"/"L"等）
  AND staff_id != ''
  AND zhsf_cents > 0  -- 排除负值（撤单）
GROUP BY year, month, org;
```

### 3.2 分母：在职人力（开放问题 Q1）

| 选项 | SQL |
|------|-----|
| A：月末在职（推荐） | `SUM(hc_end)` |
| B：月均在职 | `(SUM(hc_start) + SUM(hc_end)) / 2.0` |
| C：月初在职 | `SUM(hc_start)` |

**默认用 A（月末）**，待用户确认。

### 3.3 维度切分

按 `fact_headcount` 的 `org / sales_seq / rank_grade / tenure_band / biz_mode` 维度分组。

## 4. 计算口径

### 4.1 核心 SQL

```sql
WITH active AS (
  SELECT p.year, p.month, h.org, h.sales_seq,
         COUNT(DISTINCT p.staff_id) AS act_n
  FROM fact_premium p
  JOIN fact_headcount h
    ON h.staff_id = p.staff_id
   AND h.year = p.year
   AND h.month = p.month
  WHERE p.term_type = '长期险'
    AND p.staff_id != ''
    AND p.zhsf_cents > 0
  GROUP BY p.year, p.month, h.org, h.sales_seq
),
total AS (
  SELECT year, month, org, sales_seq,
         SUM(hc_end) AS hc_n
  FROM fact_headcount
  GROUP BY year, month, org, sales_seq
)
SELECT t.year, t.month, t.org, t.sales_seq,
       COALESCE(a.act_n, 0) AS active_n,
       t.hc_n AS total_n,
       1.0 * COALESCE(a.act_n, 0) / NULLIF(t.hc_n, 0) AS activity_rate
FROM total t
LEFT JOIN active a USING(year, month, org, sales_seq)
ORDER BY t.year, t.month, t.org;
```

**注意**：
- 用 `LEFT JOIN`，避免某机构当月零活动时丢行
- 分母 `NULLIF` 保护，避免除 0
- 出单时用工号关联人力表（同月），跨月不算
- 分子要求 `zhsf_cents > 0` 过滤撤单

### 4.2 多月趋势

按 `period_ym` 排序后展开为折线图。

### 4.3 同比

```
yoy = (curr_rate - base_rate) × 100  (绝对差，单位 pp)
```

> 活动率本身是百分比，同比用"百分点差"更直观，不用相对增速。

### 4.4 活动率分布

按 `org` 分组后，用 `(active_n, total_n)` 计算每个机构的活动率，绘制直方图（横轴活动率、纵轴机构数）。

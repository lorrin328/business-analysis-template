## 2. 范围

### 2.1 包含

- 月度在职人力趋势（折线 / 柱状）
- 多维度对比：按机构、销售序列、职等、司龄区间、**外勤管理职 vs 一线营销员**
- 月度增员/减员折线（按工号差额推算）

## 4. 计算口径

### 4.1 月末在职人力

```sql
SELECT period_ym, org, SUM(hc_end) AS headcount
FROM fact_headcount
WHERE period_ym BETWEEN ? AND ?
  AND <全局筛选>
GROUP BY period_ym, org
ORDER BY period_ym;
```

### 4.2 月初/月末/月均人力（可切换口径）

```sql
SELECT period_ym, org,
       SUM(hc_start) AS hc_start_total,
       SUM(hc_end) AS hc_end_total,
       (SUM(hc_start) + SUM(hc_end)) / 2.0 AS hc_avg
FROM fact_headcount
GROUP BY period_ym, org;
```

> 模块 04（活动率）默认使用 `hc_avg`；模块 03（本模块）三口径均提供切换。

### 4.3 ~~留存率~~（v1 移除）

> ❌ 用户 2026-04-26 决策：留存率本期不做，后续再定义。
>
> 待 v2 重新讨论时，候选定义：
> - 简化：`月末在职 / 月初在职`
> - 严格（13 月留存）：`(本月在职 ∩ 13 月前在职) / 13 月前在职`
> - 同期保留率：`(月末在职 ∩ 月初在职) / 月初在职`

### 4.4 增员/减员（推算）

由于没有精确入司/离司日期，按月度差额推算：

```sql
-- 月度净变化
SELECT a.period_ym, a.org,
       SUM(a.hc_end) - LAG(SUM(a.hc_end)) OVER (PARTITION BY a.org ORDER BY a.period_ym) AS net_change
FROM fact_headcount a
GROUP BY a.period_ym, a.org;
```

更精细的"新增 vs 流失"（基于工号比对）：

```sql
-- 当月新增 = 月末在职但上月不存在
WITH curr AS (SELECT period_ym, staff_id, hc_end FROM fact_headcount),
prev AS (SELECT period_ym, staff_id, hc_end FROM fact_headcount)
SELECT c.period_ym,
       SUM(CASE WHEN c.hc_end = 1 AND p.hc_end IS NULL THEN 1 ELSE 0 END) AS hires,
       SUM(CASE WHEN c.hc_end = 0 AND p.hc_end = 1 THEN 1 ELSE 0 END) AS leavers
FROM curr c
LEFT JOIN prev p ON p.staff_id = c.staff_id
                 AND p.period_ym = strftime('%Y-%m', date(c.period_ym || '-01', '-1 month'))
GROUP BY c.period_ym;
```

> ⚠️ 工号变更视为不同人（用户决策）：员工换工号后，旧工号视为流失、新工号视为新增。

### 4.5 司龄结构

```sql
SELECT period_ym, tenure_band, SUM(hc_end) AS n
FROM fact_headcount
GROUP BY period_ym, tenure_band
ORDER BY period_ym, tenure_band;
```

司龄区间已经分好（a-f 6 档），直接 GROUP BY。

### 4.6 入司分析

```sql
SELECT hire_year, hire_month, COUNT(DISTINCT staff_id) AS hire_n
FROM fact_headcount
WHERE period_ym = ?  -- 取最新月份
GROUP BY hire_year, hire_month
ORDER BY hire_year, hire_month;
```

### 4.7 外勤管理职聚合（新增）

```sql
-- 外勤管理职 vs 一线营销员 月度对比
SELECT period_ym,
       mgmt_level,  -- '外勤管理职' / '一线营销员'
       SUM(hc_end) AS hc_n
FROM fact_headcount
GROUP BY period_ym, mgmt_level
ORDER BY period_ym, mgmt_level;
```

`mgmt_level` 派生规则待用户在 P3 启动前提供（见主方案 Q9）。临时占位规则：
- 外勤管理职：`rank_grade` 或 `rank_level` 值含「主管/经理」关键字
- 一线营销员：其他

## 5. 增量导入策略（用户 2026-04-26 决策）

### 5.1 工作流

用户后续陆续补充历史月份 N1AI-人力基表，需要：

1. 用户点击"导入人力数据"按钮
2. 系统识别文件中包含哪些 `period_ym`（统计年+统计月）
3. 系统检测每个 `period_ym` 当前 `fact_headcount` 是否已有数据
4. 处理策略：

| 场景 | 策略 |
|------|------|
| 新月份（库中无） | 直接 INSERT |
| 已有月份（重导入修正） | 删除该月数据后 INSERT（使用事务） |
| 文件含多月，部分新部分旧 | 逐月处理，按上述规则 |

### 5.2 SQL 实现

```sql
-- 导入前清空目标月（防重复）
DELETE FROM fact_headcount WHERE period_ym IN (?, ?, ...);

-- 然后批量 INSERT
INSERT INTO fact_headcount (...) VALUES (...), (...), ...;
```

### 5.3 唯一索引

```sql
CREATE UNIQUE INDEX uq_headcount_staff_month
ON fact_headcount(period_ym, staff_id);
```

防止同月同工号重复（小计行已在导入前过滤）。

### 5.4 用户提示

导入完成后，UI 提示：
- 本次导入：N 行
- 覆盖已有月份：YYYY-MM, YYYY-MM
- 当前库中包含月份：YYYY-MM ~ YYYY-MM
- 跳过小计行：4 行（如有）

## 1. 背景

"队伍产能"是反映人均产出的核心指标。用户原话："未来要对队伍产能做定义"——说明定义本身需在本模块中确定。

## 2. 范围

### 2.1 包含

- 多种产能定义切换（人均保费、人均件数、活跃人均、合格人均等）
- 月度产能趋势
- 跨机构对比、同比对比
- 司龄分层产能（不同司龄段的人均产出）
- 产能 × 活动率联合视图（散点图：x = 活动率, y = 产能）

### 2.2 不包含

- 个人级产能排名（隐私）
- 产能预测（v2+）

## 3. 数据需求

依赖模块 03 的 `fact_headcount` + 现有 `fact_premium`。

## 4. 产能定义讨论（核心）

下面列举常见的"队伍产能"定义口径。**用户需选定一个或多个作为本模块的展示指标**。

### 4.1 候选定义

| 编号 | 定义 | 公式 | 优点 | 缺点 |
|------|------|------|------|------|
| **D1** | 人均保费（含未出单） | 总保费 / 在职人力 | 直观、含全员 | 受活动率影响大 |
| **D2** | 活跃人均保费 | 总保费 / 出单人数 | 反映活跃员工真实水平 | 未反映团队整体 |
| **D3** | 人均件数 | 总件数 / 在职人力 | 反映成单频率 | 件均价值差异大 |
| **D4** | 活跃人均件数 | 总件数 / 出单人数 | 同 D3，聚焦活跃 | 同 D2 |
| **D5** | 合格人均保费 | 长险保费 / 长险出单人 | 排除短险噪音 | 短险贡献被忽略 |
| **D6** | 司龄校准产能 | 各司龄段加权后的人均 | 公平对比新老团队 | 计算复杂 |

### 4.2 推荐组合

| 主指标 | 辅助指标 |
|--------|---------|
| **D1 人均保费**（最易理解，作为主图） | D2 活跃人均（看潜力） |
| | D3 人均件数（看频次） |
| | D5 合格人均保费（聚焦长险） |

### 4.3 分母选择

与活动率相同问题：月末 vs 月均 vs 月初。**默认月末**，可切换。

### 4.4 分子选择

口径切换：折算保费 / 期交保费 / 年化规保（与平台趋势一致）。

## 5. 计算口径示例

### 5.1 D1 人均保费（月末口径）

```sql
WITH premium_by_month AS (
  SELECT year, month, org,
         SUM(zhsf_cents) / 100.0 AS total_premium
  FROM fact_premium
  WHERE <筛选>
  GROUP BY year, month, org
),
hc_by_month AS (
  SELECT year, month, org, SUM(hc_end) AS hc_n
  FROM fact_headcount
  GROUP BY year, month, org
)
SELECT p.year, p.month, p.org,
       p.total_premium,
       h.hc_n,
       p.total_premium / NULLIF(h.hc_n, 0) AS per_capita_premium
FROM premium_by_month p
JOIN hc_by_month h USING(year, month, org)
ORDER BY p.year, p.month;
```

### 5.2 D2 活跃人均

```sql
SELECT year, month, org,
       SUM(zhsf_cents) / 100.0 / NULLIF(COUNT(DISTINCT staff_id), 0) AS active_per_capita
FROM fact_premium
WHERE staff_id != '' AND zhsf_cents > 0
GROUP BY year, month, org;
```

### 5.3 D6 司龄校准产能（高级，v2 考虑）

按司龄分箱后取人均，再用全队司龄结构做加权平均。

```sql
WITH per_band AS (
  -- 每个司龄段的人均
  SELECT h.year, h.month, h.org, h.tenure_band,
         SUM(p.zhsf_cents) / 100.0 / NULLIF(SUM(h.hc_end), 0) AS pc
  FROM fact_headcount h
  LEFT JOIN fact_premium p
    ON p.staff_id = h.staff_id
   AND p.year = h.year AND p.month = h.month
  GROUP BY h.year, h.month, h.org, h.tenure_band
),
benchmark_struct AS (
  -- 基准司龄结构（如全公司平均）
  SELECT tenure_band, AVG(weight) AS w FROM ...
)
SELECT ...   -- 加权平均
```

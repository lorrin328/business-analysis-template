## 1. 背景与目标

### 1.1 业务背景

网电多元条线在产品上线前，需要快速评估**不同基本法方案下的费用率水平**，以支持产品定价和利润测算决策。当前依赖两个Excel手工测算：

- **证保费用率预估.xlsx**：覆盖证保渠道新法/老法两套体系，支持不同人产水平对比
- **OTO新产品费用测算(模板).xlsx**：覆盖OTO渠道老法体系，三层管理架构（专员→TL→UM）

Excel工具存在以下痛点：
- 参数调整后需手动复制公式，容易出错
- 无法直观对比不同方案（新法vs老法、标准版vs费优版）
- 参数表分散在不同Sheet，维护困难
- 无法多人协作、版本管理混乱

### 1.2 模块目标

在经营分析看板中新增**「基本法测算」**入口，提供：

1. **参数化配置**：将Excel中的参数表转为UI可配置项
2. **实时计算**：输入产品假设后即时输出费用率
3. **方案对比**：支持新法/老法/不同人产水平的多维度对比
4. **结果导出**：支持导出测算结果表格

### 1.3 范围边界

| 在范围内 | 不在范围内 |
|----------|-----------|
| 证保渠道（新法+老法） | 经代渠道基本法 |
| OTO渠道（老法） | 蚁桥渠道基本法 |
| 单产品/单缴期测算 | 组合产品测算 |
| 费用率估算 | 实际佣金发放系统 |
| 参数可视化配置 | 参数版本历史管理（P2） |

## 2. 业务逻辑分析

### 2.1 核心概念

```
┌─────────────────────────────────────────────────────────────┐
│  基本法费用率 = 基本法成本合计 / 保费规模（人产 × 团队规模）   │
│                                                             │
│  基本法成本 = 专员成本 + 主管/经理成本                         │
│  ├─ 专员成本 = FYC/提奖 + 基本佣金 + 绩效佣金                  │
│  └─ 管理成本 = 管理津贴 + 育成津贴 + 主管基本佣金 + 分摊        │
└─────────────────────────────────────────────────────────────┘
```

**人产**：人均保费产出（万元/人/月），是测算的核心输入变量。

### 2.2 证保渠道 — 新法体系

#### 2.2.1 输入参数

| 参数 | 说明 | 默认值 | 来源 |
|------|------|--------|------|
| 人产（万元） | 人均保费产出 | 20 | 用户输入 |
| 缴期 | 趸交/3年/5年/10年 | 10年缴 | 用户选择 |
| FYC率 | 首年佣金率 | 按缴期 | 参数表 |
| 专员职级 | 初级/中级/高级/资深服务顾问等 | 按FYC匹配 | 参数表 |
| 经理职级 | 初级/高级/资深/首席经理 | 按团队FYC匹配 | 参数表 |

#### 2.2.2 计算链路

```
保费规模 = 人产 × 10000（元）
FYC(元) = 保费规模 × FYC率

专员成本：
  基本佣金 = 职级对应固定值（参数表）
  绩效佣金 = 职级对应固定值（参数表）
  专员基本法成本 = FYC + 基本佣金 + 绩效佣金

经理成本：
  经理管理津贴 = 团队FYC × 管理津贴率（如5%）
  经理育成津贴 = 团队FYC × 育成津贴率（如1.875%）
  经理基本佣金 = 经理职级对应固定值
  经理分摊基本佣金 = 经理基本佣金 ÷ 团队人数（默认4人）
  经理基本法成本 = 管理津贴 + 育成津贴 + 基本佣金 + 分摊

基本法成本合计 = 专员成本 + 经理成本
基本法费用率 = 基本法成本合计 / 保费规模
```

#### 2.2.3 职级匹配规则（新法）

| 职级 | 基本佣金 | 月均FYC要求 | 绩效佣金 |
|------|---------|------------|---------|
| 初级服务顾问 | 3,000 | 600 | 0 |
| 中级服务顾问 | 4,000 | 1,600 | 2,000 |
| 高级服务顾问 | 5,000 | 4,000 | 2,500 |
| 资深服务顾问 | 7,000 | 7,500 | 4,000 |
| 高级客户专家 | 8,000 | 9,000 | 7,000 |
| 资深客户专家 | 10,000 | 11,000 | 10,000 |
| 首席客户专家 | 12,000 | 18,000 | 23,000 |

**匹配逻辑**：根据当月FYC金额，匹配到满足"月均FYC要求"的最高职级。

经理职级同理：初级经理(6,000/6,000)、高级经理(8,100/12,000)、资深经理(10,500/28,000)、首席经理(12,000/42,000)。

### 2.3 证保渠道 — 老法体系

#### 2.3.1 输入参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 规保 | 规模保费（元） | 1,000,000 |
| 缴期 | 0(趸交)/3/5/10 | 10 |
| 险种系数 | 产品类型系数 | 按缴期 |
| 缴期系数 | 缴费年限系数 | 按缴期 |
| 名单系数 | 名单来源系数 | 1 |
| 对应职级 | 客户专员/客户经理等 | 按NFYP匹配 |

#### 2.3.2 计算链路

```
NFYP = 规保 × 险种系数 × 缴期系数 × 名单系数

专员成本：
  专员提奖 = NFYP × 提奖系数（阶梯表，按NFYP区间）
  基本佣金 = 职级对应固定值（A类/B类）
  专员基本法成本 = 提奖 + 基本佣金

主管成本：
  主管管理津贴 = 团队NFYP × 团队提奖系数
  主管基本佣金 = 主管职级对应固定值
  主管基本佣金分摊 = 主管基本佣金 ÷ 团队人数
  主管基本法成本 = 管理津贴 + 基本佣金 + 分摊

年终奖 = 专员基本法成本 × 年终奖系数（约5.5%）

基本法成本合计 = 专员成本 + 主管成本 + 年终奖
基本法费用率 = 基本法成本合计 / 规保
```

#### 2.3.3 提奖系数阶梯（老法）

| NFYP下限（含） | 初佣系数 |
|---------------|---------|
| 0 | 7% |
| 5,000 | 11% |
| 10,000 | 13% |
| 15,000 | 15% |
| 30,000 | 16.5% |
| 50,000 | 18% |
| 80,000 | 19.5% |
| 110,000 | 21% |
| 150,000 | 22.5% |
| 200,000 | 24% |
| 250,000 | 25% |

**注意**：Excel中同时存在另一套提奖系数（针对主管层级），NFYP区间和系数不同。

### 2.4 OTO渠道 — 老法体系

OTO老法与证保老法核心逻辑相同，但管理架构为**三层**：

```
专员 → TL（Team Leader，团队长） → UM（Unit Manager，经理）
```

#### 2.4.1 计算链路

```
专员成本 = 提奖 + 固定收入（同证保老法）

TL主管成本：
  TL团队提奖 = TL团队NFYP × TL团队提奖系数
  TL主管基本佣金 = TL职级对应固定值
  TL主管基本佣金分摊 = TL基本佣金 ÷ TL下属人数
  TL主管基本法成本 = 团队提奖 + 基本佣金 + 分摊

UM经理成本：
  UM团队提奖 = UM团队NFYP × UM团队提奖系数
  UM经理基本佣金 = UM职级对应固定值
  UM经理基本佣金分摊 = UM基本佣金 ÷ UM下属团队数
  UM经理基本法成本 = 团队提奖 + 基本佣金 + 分摊

基本法成本合计 = 专员 + TL + UM
基本法费用率 = 合计 / 规保
```

#### 2.4.2 OTO经理职级参数

| 职级 | 基本佣金 | 月均NFYP要求 |
|------|---------|------------|
| 储备经理 | 6,000 | 180,000 |
| 见习经理 | 9,000 | 300,000 |
| 初级经理 | 10,400 | 300,000 |
| 中级经理 | 11,700 | 450,000 |
| 高级经理 | 12,900 | 650,000 |
| 资深经理 | 15,000 | 850,000 |
| 总监 | 19,000 | 1,200,000 |

### 2.5 方案对比（汇总表逻辑）

## 4. 交互设计

### 4.1 状态流转

```
[初始态]
  │
  ▼
[输入参数] ──→ 人产/缴期/方案选择
  │
  ▼
[点击测算] ──→ 实时计算
  │
  ▼
[结果展示] ──→ 表格+费用率高亮
  │
  ├─→ [切换缴期] ──→ 更新单缴期结果
  ├─→ [高级选项] ──→ 展开/折叠高级参数
  ├─→ [导出Excel] ──→ 下载结果表格
  ├─→ [对比模式] ──→ 新法vs老法并排对比
  └─→ [参数配置] ──→ 打开参数面板
```

### 4.2 关键交互细节

| 场景 | 交互 |
|------|------|
| 人产输入变化 | 实时重新计算（防抖300ms） |
| 缴期切换 | 更新FYC率/险种系数显示，重新计算 |
| 方案切换（新法↔老法） | 左侧输入区字段变化，保留人产 |
| 考核达成率 < 100% | 基本佣金按达成比例打折，实时更新 |
| 费用率 > 30% | 红色高亮警告 |
| 费用率 < 5% | 绿色高亮（低成本） |

### 4.3 对比模式

点击「对比模式」后，界面变为并排布局：

```
┌─────────────────────┬─────────────────────┐
│ 方案A：标准版（新法） │ 方案B：费优版（新法） │
│ 人产：20万          │ 人产：20万          │
│ 缴期：10年          │ 缴期：10年          │
│ ...                 │ ...                 │
├─────────────────────┼─────────────────────┤
│ 专员成本：24,000     │ 专员成本：24,000     │
│ 经理成本：7,575      │ 经理成本：7,575      │
```

差异项用颜色标注（绿色=降低，红色=升高）。

### 6.3 关键算法

**职级匹配算法**：

```javascript
function matchLevel(fyc, levels) {
  // 按月均FYC要求降序排列，找到第一个满足条件的职级
  const sorted = [...levels].sort((a, b) => b.monthlyFycReq - a.monthlyFycReq);
  for (const level of sorted) {
    if (fyc >= level.monthlyFycReq) return level;
  }
  return sorted[sorted.length - 1]; // 最低职级兜底
}
```

**阶梯系数查询**：

```javascript
function lookupLadder(value, ladder) {
  const sorted = [...ladder].sort((a, b) => b.threshold - a.threshold);
  for (const step of sorted) {
    if (value >= step.threshold) return step.rate;
  }
  return ladder[0].rate;
}
```

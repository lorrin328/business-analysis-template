# v1.0.79 发布说明

## 结论

本版本修复 Web 上传 Excel 后数据仍不更新的底层原因：原始明细表字段结构与新 Excel 字段不一致时，导入事务会被回滚，导致聚合表继续停留在旧截止日。

## 根因

当前 4 份 Excel 可以通过部署重建脚本正确生成新聚合数据，但通过页面上传时，`write_raw_table_incremental` 在写入 `performance` 原始表时发现新 Excel 字段多于数据库已有字段，抛出 `schema mismatch`，整次事务回滚。

因此用户看到的不是新数据计算错误，而是新数据没有真正写入运行库。

## 修复内容

1. 原始明细表遇到新字段时自动扩展列。
2. 扩展列后继续按年月删除同期间旧 raw 数据并追加新 raw 数据。
3. 聚合表、原始表、导入记录在同一事务内成功提交。
4. 保留历史 raw 数据，不再因为 Excel 新增字段要求整库重建。

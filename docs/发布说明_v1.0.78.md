# v1.0.78 发布说明

## 结论

本版本针对“部署并重新上传 Excel 后页面仍显示旧口径/旧截止日数据”的风险，补齐接口防缓存和服务器侧导入诊断能力。

## 修复内容

1. **接口防缓存**
   - 所有 `/api/` 业务接口返回 `Cache-Control: no-store, no-cache, must-revalidate, max-age=0`。
   - 前端 `fetchJson` 请求同步使用 `cache: no-store` 和 `Cache-Control: no-cache`。

2. **上传后强制刷新服务端数据**
   - Excel 上传成功后生成新的刷新标识。
   - KPI、机构维度、平台趋势、交期结构等后续读取请求不再复用旧接口响应。

3. **新增导入诊断接口**
   - 管理员可访问 `/api/diagnostics/import-status?year=2026`。
   - 返回运行数据库路径、最近导入记录、转型/经代日级截止日、转型年度累计、机构日累计及 5月26日至31日的分日对账值。

## 业务核验口径

当前用户反馈转型年度累计应在 `16025` 万左右。以项目根目录 4 份 Excel 隔离重建验证，转型机构日累计在 5月27日约为 `16025.3158` 万；若页面显示约 `15958` 万，通常意味着页面/API 仍取到了截至 5月26日的旧响应或旧运行库。

本版本不调整指标公式，重点保证页面读取的是最新服务端结果，并提供服务器端可核验依据。

# v1.0.43 发布说明

## 核心修复

本版本修复代码审计中确认的问题：部署环境版本保护、产品结构规模保费查询、原始明细增量导入删除，以及单侧日级数据时的 KPI 截止日回退。

## 本次调整

1. 部署保护
   - `deploy/deploy.sh` 在创建虚拟环境前检测 `python3 >= 3.10`。
   - 旧系统 Python 不满足要求时提前退出，并提示使用 Docker 或升级 Python。

2. 产品结构 `gm` 口径
   - 产品结构 raw 查询不再直接拼接可能不存在的 SQLite 双引号列名。
   - 转型业务按实际存在列优先选择 `年化规保`，其次 `规模保费`。
   - 经代业务按实际存在列优先选择 `承保年化规保`，其次 `年化规保`、`规模保费`。

3. 原始表增量导入
   - 删除同月旧 raw 行时统一规整 `-`、`/`、`.`、`年`、`月`、`日` 和空格。
   - 避免 `2026.05.01`、`2026年05月01日` 这类日期格式导致旧明细未删除。

4. KPI 截止日策略
   - 转型与经代两侧均有日级数据时，继续按各自真实日级截止日计算 KPI。
   - 只有单侧有日级数据时，不再混用半月日表和另一侧整月月表，而是回退到最新日级月份之前的完整月份。
   - 返回的 `daily_cutoff.policy` 增加 `mode`、`partial_daily`、`fallback_month`，便于前端和诊断识别当前策略。

5. 机构分析日级截取
   - 机构分析年度期交保费按渠道分别使用当前年度日级 cutoff。
   - 避免全局最新日来自 OTO 时，证保/蚁桥去年同期也被截到更晚日期。

## 验证

- SQLite 复现实验确认缺少 `年化规保` 时，`规模保费` 可正确参与产品结构 `gm` 统计。
- SQLite 复现实验确认点号日期和中文日期可匹配同月删除。
- 本机系统 Python 为 3.9.6，无法运行当前后端测试套件；部署脚本已显式拦截该环境。

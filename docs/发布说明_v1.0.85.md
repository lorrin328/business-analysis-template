# v1.0.85 发布说明

## 发布类型

修复版本：操作日志北京时间展示修复。

## 问题原因

Ubuntu 系统时间和时区正常：

- 本地时间：Asia/Shanghai / CST / UTC+8
- UTC 时间比北京时间少 8 小时

操作日志显示为凌晨 1 点，是因为 SQLite 的 `CURRENT_TIMESTAMP` 默认写入 UTC，前端此前直接展示数据库原始值，没有转换为北京时间。

## 核心修复

1. 操作日志接口返回时，将 `created_at` 从 UTC 转换为北京时间。
2. 同时新增 `created_at_utc` 字段，保留原始 UTC 时间，便于后续审计追溯。
3. 前端操作日志说明改为“时间按北京时间展示”。

## 验证

- 新增操作日志时区转换测试。
- 权限与前端定向测试通过。
- 全量测试通过。

# v1.0.76 发布说明

## 发布类型

fix / import / cache-busting

## 核心结论

本版本继续修复“导入 Excel 后看板数据不变化”的链路风险。v1.0.75 已让前端上传带 `force=true`，但主页面没有给 `upload.js` 加版本号，浏览器可能继续使用旧上传脚本。本版本同时从前端缓存和后端默认行为两侧兜住，确保用户重新导入 Excel 后会真实重写数据。

## 主要变更

1. 主页面加载 `js/upload.js?v=1.0.76`，强制浏览器获取新上传脚本。
2. 后端 `/api/upload` 和 `/api/import` 默认 `force=true`。
3. 即使请求没有显式带 `force=true`，重复 Excel 文件也会重新解析、重写原始表和聚合表。
4. 导入接口返回值和操作日志增加 `force` 标识，便于排查服务器导入链路。
5. 补充自动化测试，覆盖上传脚本版本号、默认强制导入和前端上传强制参数。

## 验证重点

- 页面 HTML 中 `upload.js` 必须带 `v=1.0.76`。
- 导入接口默认 `force=true`。
- 导入后 `operation_logs.detail.force` 应为 `true`。
- 导入完成后 KPI、机构维度等数据应重新从服务端接口加载。

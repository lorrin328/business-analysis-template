# 经营分析看板 - 太平人寿网电多元条线

**太平人寿保险有限公司网电多元条线** 保险经营分析单页看板。

> 📚 **完整文档体系**：本 README 描述当前实现现状。架构演进、模块化方案、新模块需求、数据规范请见 [docs/](./docs/README.md)。

## 项目概览

本项目定位为太平人寿网电多元条线经营分析看板/经营驾驶舱，当前版本为 **v1.0.41**，采用 **FastAPI + SQLite + 原生 HTML/JS + ECharts** 的轻量架构。核心目标是保证数据处理准确、指标口径统一、页面展示清晰、多人访问稳定，并为后续扩展预留空间。

本项目覆盖以下业务线：

| 业务线 | 说明 |
|--------|------|
| **OTO** | 线上到线下融合业务 |
| **证保** | 证券渠道保险业务 |
| **蚁桥** | 蚂蚁渠道桥梁业务（网服） |
| **经代** | 线上保险经纪/代理业务 |

**当前重点**：正在推动 **职域营销** 新模式，主要与 **团险渠道** 合作开展，目前处于 **试点阶段**。

## 核心模块

### 1. KPI 概览
- 期交保费达成率（整体 / 经代 / 转型 / OTO / 证保 / 蚁桥）
- 价值达成率
- 活动率（同比）
- 人均保费 / 人均产能

### 2. 业务分析
- **业务平台趋势**：支持年度/季度/月度维度，多业务系列复选加总，三种保费口径切换；季度、月度统一按日累计趋势展示，并加入上一年度同期同口径趋势线
- **产品结构**：饼图展示各产品分类占比

### 3. 机构维度
- 支持机构复选、年度/季度/月度切换
- 按机构 × 业务模式展示期交保费、价值保费、10年期产品、商保年金、保障类产品
- 实际、目标、达成率、同比均按所选期间同口径计算

### 4. 队伍分析
- **队伍趋势**：人力规模、活动率、人均保费、人均产能
- 支持年度/季度维度，OTO/证保/蚁桥复选

## 当前架构

> **运行入口边界**：当前生产页面只加载根目录 `经营分析模板.html` 与 `js/api-client.js`。`js/` 目录下其他模块文件保留为模块化迁移参考，未被生产 HTML 显式加载前，不应视为运行时代码；如后续接入，必须同步更新 HTML `<script>` 列表并通过前端静态测试。

```text
Excel 基表
  -> FastAPI 后端导入
  -> 字段转换与数据校验
  -> SQLite 聚合表与目标表
  -> 统一 API
  -> 经营分析模板.html + ECharts 展示
```

主要目录：

| 目录 | 用途 |
|---|---|
| `backend/api/` | 统一 API 路由 |
| `backend/metrics/` | 指标公式中心 |
| `backend/config/` | 业务线、指标、机构、运行配置 |
| `backend/services/` | 数据转换、查询服务、响应格式 |
| `backend/validators/` | 导入、目标、机构校验 |
| `deploy/` | Ubuntu 部署文件 |
| `tests/` | 基础验收测试 |

## 数据流程

```
Excel 基表（4个文件）
    ↓  Python 解析 + 聚合
SQLite 数据库（backend/business_data.db）
    ↓  FastAPI 接口
生产 HTML 动态读取（platformMock + teamMock + 机构聚合）
    ↓  浏览器渲染
ECharts 图表
```

### 数据源

| 文件名 | 内容 | 行数 |
|--------|------|------|
| `AI-经营分析业绩基表_*.xlsx` | 转型业务保单明细（OTO/证保/蚁桥） | ~69,000 |
| `AI-经营分析价值基表_*.xlsx` | 价值保费数据 | ~140 |
| `N1AI-人力基表_*.xlsx` | 人力数据（在职/活动人力） | ~2,500 |
| `经代业绩分析.xlsx` | 经代渠道业绩数据 | ~125,000 |

### 数据更新步骤

1. 启动 FastAPI 服务。
2. 通过页面上传 Excel，或调用 `/api/import`。
3. 后端完成解析、校验、聚合并写入 `backend/business_data.db`。
4. 页面通过 API 重新读取数据并刷新 KPI、平台趋势、机构、队伍和产品图表。

## 目标设置

目标数据已迁移为服务端优先：

1. 目标设置功能已合并入 `经营分析模板.html`，不再维护独立 `目标设置.html`。
2. 当前生产页面保存目标时优先调用统一接口 `/api/targets?year=YYYY`。
3. 后端同时写入 `target_config` JSON 配置和 `target_values` 行表。
4. `localStorage` 仅作为后端不可用时的降级缓存，不作为正式多人共享数据源。
5. 修改目标后可通过页面“重新计算”刷新相关 KPI 和图表。

支持指标：
- 期交保费
- 价值保费
- 商保年金
- 保障类产品
- 10 年期产品

每个指标分年度/季度/月度三个维度，支持整体/经代/转型/OTO/证保/蚁桥六个口径。

## 开发说明

### 业务口径映射

Excel 原始口径 → 看板口径：

| Excel 口径 | 看板口径 |
|-----------|---------|
| 证券 | 证保 |
| 网服 | 蚁桥 |
| OTO | OTO |

### 保费类型定义

- **期交保费**：非趸交的长期险规模保费 + 短期险规模保费 + 长险趸交规模保费 × 0.1
- **规模保费（年化规保）**：各类产品的规模保费
- **折算保费**：长险趸交 × 0.1 + [3年,5年) × 0.3 + [5年,10年) × 0.5 + 10年及以上 × 1 + 短险 × 0.1

### 指标计算

- **活动率** = 活动人力 / 在职人力 × 100%
- **人均保费** = 新单保费 / 月均在职人力
- **人均产能** = 新单保费 / 月均活动人力
